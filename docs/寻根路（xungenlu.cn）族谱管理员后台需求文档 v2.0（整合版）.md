寻根路（xungenlu.cn）族谱管理员后台需求文档 v2.0（整合版）

文档版本：v2.0  

对应系统版本：寻根路 GeneaSphere Platform v1.0  

面向对象：开发团队、测试团队、产品经理  

修订说明：v2.0 整合了之前所有分散的家族管理员功能需求（短信、归宗合并、家庭关系更新、照片标注、寻找小伙伴、直系血缘视频、家庭图册、迁徙地图、工具箱、邀请二维码等），并补充了遗漏的管理功能（家族公告、数据统计看板、批量操作、版本管理、回收站等），形成一份完整的族谱管理员后台需求。

1. 产品概述

1.1 背景

族谱管理员（通常为族长、修谱负责人）需要在一个统一的后台管理本家族的所有事务。此前功能分散在多个需求文档中，管理员操作时需要频繁切换页面。本需求将所有功能整合到一个清晰的后台界面，提升管理效率。

1.2 目标

• 提供一站式家族管理后台，覆盖成员管理、内容审核、寻亲归宗、系统设置、印刷订单、短信通知、数据统计等全部功能。

• 保持操作路径简洁，关键数据一目了然。

• 确保多角色权限隔离（Owner/Admin/Editor/Viewer）。

2. 角色与权限

角色 权限范围

家族超级管理员（Owner） 全部权限，可转让所有权，不可被删除

家族管理员（Admin） 内容审核、寻亲处理、系统设置、成员管理（不可删除 Owner）

编辑者（Editor） 增删改人员信息、上传照片、修改生平、管理个人相册

观察员（Viewer） 仅查看族谱、时光长廊、迁徙地图等，无编辑权限

注：本后台仅面向 Owner 和 Admin，Editor 和 Viewer 无后台入口。

3. 功能模块

3.1 控制台（Dashboard）

• 关键指标：家族总人数、在世人数、照片总数、存储用量、短信余额、待审核内容数、待处理寻亲申请数、本月印刷订单数。

• 待办事项：最近 5 条待审核内容（影像/寻亲帖）、待处理认亲申请，点击直接跳转。

• 快速入口：一键跳转“隐私配置”、“字辈管理”、“发送短信”。

3.2 成员管理

3.2.1 成员列表

• 展示所有家族成员，字段：头像、姓名、性别、世代、房支、角色、手机号（脱敏）、最后活跃时间。

• 支持按世代、房支、角色筛选，关键字搜索。

• 点击姓名跳转至该成员的个人详情页（前台）。

3.2.2 权限分配

• 点击“权限”按钮弹出对话框，下拉选择角色（Owner/Admin/Editor/Viewer）。

• 敏感操作：将 Admin 降级为 Viewer 需二次确认（输入当前管理员密码）。

• 转让 Owner：仅 Owner 可操作，需输入新 Owner 用户名并确认，原 Owner 自动降为 Admin。

3.2.3 批量操作

• 支持勾选多个成员，批量修改角色、批量发送短信、批量导出信息（CSV）。

3.3 内容审核

3.3.1 影像审核

• 展示本家族待审核的照片/视频（用户上传或 AI 标记）。

• 支持缩略图预览，点击查看大图。

• 操作：通过 / 驳回（填写理由）/ 设为封面（优质照片可设为聚落封面）。

3.3.2 生平审核

• 展示待审核的生平传记，支持富文本预览。

• 操作：通过 / 驳回。

3.3.3 寻亲帖审核

• 展示本家族发布的寻亲帖，支持编辑、下架、置顶。

3.3.4 举报管理

• 查看本家族成员被举报的内容，处理：确认违规（删除内容并警告）或驳回举报。

3.4 寻亲与归宗管理

3.4.1 认亲申请列表

• 展示申请加入本家族的请求，字段：申请人、祖籍地、字辈、匹配度、申请时间、状态。

• 点击查看详情，系统自动比对重叠信息（字辈、地名、人物）。

3.4.2 归宗合并

• 通过申请后，进入归宗合并向导：

  • 第一步：选择挂载点（主家族树中的已故人员）。

  • 第二步：确认世代偏移（系统自动推算，可微调）。

  • 第三步：执行合并（自动创建快照，支持 24 小时内回滚）。

• 详细逻辑见《归宗合并功能需求文档》。

3.4.3 寻亲帖管理

• 管理本家族发布的寻亲帖，支持编辑、下架、置顶。

3.5 系统设置

3.5.1 隐私配置

• 游客可见范围：开关“允许游客查看已故人员信息”、数字输入“仅展示前 N 代”、开关“隐藏在世人员照片/配偶信息”。

• 亲属验证登录：开启/关闭，选择验证问题类型（父亲姓名、祖父出生年份等），设置容错次数。

3.5.2 字辈管理

• 展示当前字辈序列，支持增删改。

• 新增人员时自动校验姓名是否含对应字辈，若不匹配则警告（不强制阻止）。

3.5.3 家庭关系更新审核

• 查看本家族成员提交的家庭关系变更申请（婚姻状态、子女归属等）。

• 操作：通过 / 驳回 / 标记为需线下确认。

• 变更历史可追溯。

3.5.4 云存储

• 展示本家族存储用量（饼图：照片/视频/其他），剩余容量。

• 扩容入口（跳转至平台充值）。

3.6 印刷订单管理

• 展示本家族所有印刷订单：订单号、下单时间、规格、数量、金额、状态（待支付/印刷中/已发货/已完成/已退款）、物流单号。

• 操作：查看详情、再次购买（复刻上次配置）。

3.7 短信通知

3.7.1 发送短信

• 选择收件人：按房支、世代、地区、年龄筛选，或手动勾选。

• 编辑内容：支持插入变量（{族名称}、{收件人姓名}、{发送日期}），实时显示字数。

• 选择签名（预设模板）。

• 发送方式：立即发送 / 定时发送。

• 发送前预览，确认后提交。

• 费用：从家族余额中按成功条数扣费（0.05元/条），余额不足时弹出充值提示。

3.7.2 余额管理

• 显示当前余额、累计充值、累计消费。

• 充值入口（支持微信/支付宝，档位：50/100/200/500元，赠送比例不同）。

• 充值记录、扣费记录列表。

3.7.3 发送记录

• 历史发送记录：时间、内容摘要、收件人数、成功数、失败数、费用、状态。

• 点击详情查看失败原因（号码无效、欠费等）。

3.8 家族公告（新增）

3.8.1 发布公告

• 管理员可发布家族公告（如祭祖通知、族谱更新通知）。

• 公告类型：普通 / 置顶（最多 3 条置顶）。

• 公告内容支持富文本和图片。

• 发布后，所有家族成员在登录后的首页或族谱树页面顶部看到公告栏。

3.8.2 公告管理

• 列表展示所有历史公告，支持编辑、下架、删除。

3.9 数据统计（新增）

3.9.1 家族数据看板

• 人口统计：按世代、房支、性别分布图表。

• 照片统计：按年份、地点分布。

• 迁徙统计：迁徙事件数量、主要迁徙路线。

• 工具使用统计：AI 工具箱各工具使用次数、音像墙生成数量。

3.9.2 导出

• 支持导出成员列表（CSV）、照片列表（CSV）、迁徙事件列表（CSV）。

3.10 日志审计

3.10.1 操作日志

• 记录所有关键操作：时间、操作人、操作类型、对象、详情。

• 操作类型包括：修改关系、删除人员、通过审核、驳回审核、合并支系、修改隐私配置、修改角色、发送短信、发布公告等。

• 支持按操作人、操作类型、时间范围搜索。

• 日志不可删除，保留至少 180 天。

3.10.2 登录日志

• 记录管理员登录记录：时间、IP、设备、成功/失败。

3.11 回收站（新增）

3.11.1 已删除成员

• 显示被删除的成员列表（软删除），支持恢复（恢复后数据完整）或永久删除。

• 永久删除后不可恢复。

3.11.2 已删除内容

• 显示被删除的照片、留言、公告等，支持恢复或永久删除。

4. 界面布局

4.1 通用布局

5. 数据接口

所有接口需校验管理员 Token 和 clan_id 归属。主要接口分类：

模块 接口示例 说明

控制台 GET /admin/dashboard 统计数据

成员 GET /admin/members 成员列表

PATCH /admin/members/:id/role 修改角色

审核 GET /admin/reviews/media 待审核影像

POST /admin/reviews/media/:id/approve 通过审核

寻亲 GET /admin/searches/applications 认亲申请

POST /admin/searches/applications/:id/merge 归宗合并

设置 GET /admin/settings/privacy 隐私配置

PUT /admin/settings/privacy 更新隐私

短信 POST /admin/sms/send 发送短信

GET /admin/sms/balance 余额查询

公告 POST /admin/announcements 发布公告

日志 GET /admin/logs 操作日志

回收站 GET /admin/trash/members 已删除成员

POST /admin/trash/members/:id/restore 恢复成员

6. 非功能性需求

• 性能：列表页加载 < 2s，操作响应 < 500ms。

• 安全：所有接口校验角色权限；敏感操作二次确认；日志不可篡改。

• 多租户隔离：所有接口强制校验 clan_id 与当前管理员所属家族一致。

• 可用性：99.5% 正常运行时间。

7. 验收标准

1. Owner 登录后可看到控制台所有数据，待办事项准确。
2. Admin 可审核影像、寻亲帖，操作后前端生效。
3. 归宗合并功能正常执行，合并后数据正确，支持 24 小时内回滚。
4. 短信发送成功扣费，余额不足时提示充值。
5. 发布公告后，所有家族成员在前台看到公告。
6. 回收站中可恢复已删除成员，恢复后数据完整。
7. 所有关键操作记录在日志中，不可删除。

补充：PDF导入管理模块（新增）

功能描述

家族管理员可查看本家族成员使用PDF导入（扫描件OCR）的记录，管理OCR余额，设置家族级别的OCR免费额度。

具体功能点

1. OCR使用记录列表
   • 展示所有通过本家族账号发起的OCR导入记录。

   • 字段：发起人、文件名、页数、识别字数、费用、状态（成功/失败）、时间。

   • 支持按日期、发起人筛选。

2. OCR余额管理
   • 显示家族OCR专用余额（与短信余额独立或共用，取决于设计）。

   • 充值入口（与短信余额共用或独立）。

   • 设置月度免费额度（默认10页/1000字，管理员可调整）。

3. 失败记录重试
   • 对识别失败的PDF，管理员可点击“重新识别”按钮，系统重新调用OCR服务。

4. OCR设置
   • 开关：允许/禁止本家族成员使用OCR导入。

   • 默认免费额度配置。

补充：邀请二维码管理模块（新增）

功能描述

管理员可生成邀请二维码，查看扫码记录，管理二维码有效期。

具体功能点

1. 生成二维码
   • 选择二维码类型：家族邀请码（扫码后加入本家族）或验证码（用于族人互验）。

   • 设置有效期（默认7天，可自定义）。

   • 生成后显示二维码图片，支持下载PNG或复制短链接。

2. 扫码记录
   • 列表展示：二维码、扫码者微信昵称、扫码时间、是否已加入、验证状态。

   • 支持按二维码筛选。

3. 二维码管理
   • 查看所有已生成的二维码，支持手动失效（禁用）。

补充：迁徙事件管理（原需求已提及但未放入后台）

功能描述

管理员可在后台手动添加、编辑、删除迁徙事件，补充系统自动提取的不足。

具体功能点

1. 迁徙事件列表
   • 展示所有迁徙事件（系统自动生成+手动添加）。

   • 字段：迁出地、迁入地、时间、原因、关联人物。

   • 支持按时间、地点筛选。

2. 新增/编辑迁徙事件
   • 表单：迁出地、迁入地（支持地图选点）、时间（年份，可模糊）、原因（战乱/经商/仕宦/垦荒/其他）、描述、关联人物（从族谱中选择）。

   • 提交后自动更新迁徙地图。

3. 删除迁徙事件
   • 删除后迁徙地图相应路径消失（需二次确认）。

补充：AI工具箱使用记录（新增）

功能描述

管理员可查看本家族成员使用AI工具箱的详细记录，用于统计和成本控制。

具体功能点

2. 统计概览
   • 显示本月各工具使用次数、总消耗次数、剩余免费次数。

管理员可一键导出本家族全部数据，用于备份或迁移。

具体功能点

新增模块：图片影像管理
1. 功能概述
家族管理员需要一个集中的“影像库”来管理本家族所有的图片和视频，而不仅仅是被动地审核用户上传的内容。管理员可以主动上传、编辑、归类、删除影像，确保家族影像数据的完整性和准确性。
2. 入口位置
左侧导航菜单新增一级菜单 “影像管理”，下设两个二级菜单：
影像库（浏览/搜索/编辑/删除所有影像）
上传影像（批量上传并填写元数据）
3. 功能列表
3.1 影像库（浏览与搜索）
展示方式：网格视图（默认）和列表视图可切换。
筛选条件：
按类型：图片 / 视频
按上传者：全部 / 指定成员
按时间：上传日期范围 / 拍摄年份范围
按地点：地点关键词搜索
按关联人物：选择家族成员，筛选出包含该人物的影像
按状态：已审核 / 待审核 / 已驳回
搜索框：支持按文件名、描述、地点关键词搜索。
排序：按上传时间（最新/最旧）、拍摄时间、文件大小。
3.2 影像详情与编辑
点击任意影像进入详情页，展示：
大图/视频播放器
元数据信息：
文件名、文件大小、分辨率、上传者、上传时间
拍摄地点（可编辑，支持地图选点）
拍摄时间（可编辑，精确到日）
描述（可编辑）
关联人物（可添加/移除本家族成员，支持多选）
所属相册（可移动到其他相册）
隐私级别（仅自己可见 / 全族公开 / 上下几代公开 / 平台公开 / 同地点用户公开）
操作按钮：
保存修改
删除（软删除，进入回收站）
设为封面（可用于家族封面、聚落封面）
生成缩略图（手动触发重新生成）
查看原始图（跳转到 COS 预签名 URL）
3.3 批量操作
在影像库中勾选多个影像后，支持：
批量修改元数据：统一修改地点、时间、描述、隐私级别（仅修改勾选的字段，其他保留）。
批量关联人物：为选中的所有影像添加同一个人物标签。
批量移动：移动到指定相册。
批量删除：软删除。
批量下载：打包为 ZIP 下载（需异步处理，完成后通知）。
3.4 上传影像
上传方式：支持拖拽或点击上传，单次最多 20 个文件，单文件最大 50MB（图片） / 200MB（视频）。
上传前填写元数据（可选，也可上传后在影像库中编辑）：
拍摄地点（自动补全已有 POI）
拍摄时间
描述
关联人物（从家族成员中选择）
所属相册（选择已有相册或新建相册）
隐私级别（默认继承相册设置）
上传后处理：自动生成缩略图、展示图、原始图三份，存入 COS 对应目录。上传完成后显示成功/失败列表。
3.5 相册管理
相册列表：展示所有相册（系统相册 + 用户创建相册），显示封面、名称、照片数、创建者。
创建相册：管理员可创建家族级相册（如“2025年祭祖”、“老宅旧影”），设置默认隐私级别。
编辑/删除相册：删除相册时照片移至“未分类”相册，不删除照片本身。
3.6 回收站
显示所有已删除的影像，支持恢复（恢复后回到原相册）或永久删除。
影像在回收站中保留 30 天，超时自动永久删除。

5. 权限说明
Owner / Admin：拥有影像管理的全部权限（上传、编辑、删除、批量操作）。
Editor：可上传影像，可编辑自己上传的影像，不可删除他人影像。
Viewer：无影像管理权限，仅可查看。
6. 验收标准
管理员进入影像库，可看到所有本家族影像（含其他成员上传的），支持按条件筛选。
点击影像可编辑地点、时间、描述、关联人物，保存后前端显示更新。
批量勾选 5 张照片，统一修改地点为“王家村”，保存后 5 张照片的地点均更新。
上传 3 张新照片，填写元数据，上传成功后出现在影像库中。
删除一张照片，进入回收站可看到，恢复后照片重新出现在原相册。
创建新相册，移动 2 张照片到该相册，相册内照片数量正确。
这个模块补上了之前缺失的“主动管理影像”能力，与“影像审核”形成完整闭环。
