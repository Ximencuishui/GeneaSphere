文档版本：v1.0  

对应系统版本：寻根路 v1.0  

前置依赖：数据架构 v1.0（persons 含 birth_place、death_place、迁徙事件表）、时光长廊模块  

面向对象：开发团队、测试团队、家族管理员  

1. 功能概述

1.1 背景

族谱不仅是血缘关系的记录，更是一部家族迁徙史。从“山西洪洞大槐树”到“湖广填四川”，每一次迁徙背后都是时代的烙印。现有族谱软件只能静态展示人物关系，无法直观呈现家族在时空中的流动轨迹。因此，需要构建一个动态迁徙地图，将族谱中的时间、地点信息与历史地理数据结合，让用户以“上帝视角”见证家族数百年的漂泊与扎根。

1.2 目标

• 根据族谱中人物的生卒地、迁徙事件，自动在地图上标注 POI 点并生成迁徙路径。

• 支持按时间线自动播放，背景地图随时代切换为对应的历史朝代分割图（如明朝、清朝、民国）。

• 在迁徙过程中，动态展示与当前地点相关的家族老照片（缩略图），照片周围带有光晕效果。

• 提供交互控制（暂停/播放、调速、手动拖动时间轴），让用户自主探索。

1.3 用户故事

• 作为家族长者，我希望看到祖先从何而来，一路经历了哪些地方，感受家族的艰辛与荣光。

• 作为年轻族人，我想知道“我们家的根在哪”，并通过老照片看到先辈曾经生活过的土地。

• 作为家族管理员，我想在祭祖大会上播放这段迁徙动画，增强家族凝聚力。

2. 角色与权限

角色 权限

普通成员 可查看本家族的迁徙地图，播放动画

家族管理员 可编辑迁徙事件（增删改地点、时间、关联图片）

游客 不可查看（迁徙地图属于家族内部数据）

3. 功能列表

3.1 数据准备与 POI 自动生成

3.1.1 数据来源

• 人物生卒地：persons.birth_place、persons.death_place（已有字段）。

• 迁徙事件：新增 migration_events 表，记录每次迁徙的起始地、目的地、时间、原因、关联人物。

• 家族定居点：settlements 表中的村寨坐标（已有字段）。

3.1.2 POI 生成规则

• 系统自动提取家族中所有出现过的不重复地点（精确到县级及以上）。

• 每个地点生成一个 POI 点，包含：

  • 地名、经纬度（通过地理编码 API 自动解析）。

  • 在该地生活过的族人数量、时间段。

  • 关联的家族照片数量（来自 media_archive 中 taken_location 匹配的照片）。

3.1.3 迁徙路径生成

• 按时间顺序连接所有 POI 点，形成一条或多条路径（若家族分多支迁徙，则显示多条路径，可切换查看）。

• 路径线条颜色按时间渐变（从浅色到深色，代表时间推进）。

3.2 动态播放与时间轴

3.2.1 时间轴控件

• 底部显示水平时间轴，刻度按世纪/年代划分（根据数据跨度自动调整）。

• 时间轴上标记每个 POI 点对应的时间节点。

• 用户可拖动时间指针，地图自动跳转到对应时代的 POI 和路径段。

3.2.2 自动播放

• 点击“播放”按钮，时间指针从最早时间点开始匀速前进。

• 地图视角跟随当前 POI 点自动平移、缩放（平滑过渡）。

• 播放速度可调（1x / 2x / 4x / 8x）。

3.2.3 暂停与手动探索

• 播放过程中可随时暂停，点击任意 POI 点查看详情。

• 暂停后，地图保持当前状态，用户可自由拖拽、缩放。

3.3 背景地图：历史朝代分割图（核心特色）

3.3.1 朝代图层切换

• 当地图时间轴进入某个历史时期时，背景自动叠加对应的历史疆域图（如明朝、清朝、民国初期）。

• 朝代分割图来源：使用开源历史地图数据（如 CHGIS、Harvard Map Collection）或自绘简化版。

• 切换动画：淡入淡出过渡，时长 0.5 秒，避免突兀。

3.3.2 朝代标注

• 地图右上角显示当前朝代名称、起止年份。

• 鼠标悬停显示该朝代的简要说明（如“清朝（1636-1912），中国最后一个封建王朝”）。

3.3.3 数据精度

• 优先覆盖中国主要朝代（明、清、民国、现代）。

• 对于更早的朝代（唐宋元），提供简化版轮廓，并标注“仅供参考”。

3.4 动态图片播放与光晕效果（核心交互）

3.4.1 图片关联

• 当时间指针到达某个 POI 点时，该点关联的家族老照片以缩略图形式出现在地图上该位置周围。

• 图片排列方式：环绕 POI 点呈弧形或网格状（最多同时显示 6 张，其余通过“查看更多”展开）。

3.4.2 光晕效果

• 每张缩略图外围有一圈柔和的发光光晕（CSS box-shadow 或 SVG feGaussianBlur）。

• 光晕颜色：暖金色（#FFD700），透明度 0.6，大小随图片尺寸变化。

• 光晕呼吸动画：缓慢脉动（scale 1.0 ↔ 1.05，周期 2s），营造“历史温度”感。

3.4.3 图片交互

• 点击缩略图：弹出大图预览（复用 ImagePreview 组件）。

• 图片下方显示拍摄年份和简短描述（取自 media_archive.description）。

3.5 补充完善点（用户未提及但必要的设计）

3.5.1 多支系迁徙对比

• 若家族有多个分支（如长房迁东北、二房迁四川），地图上以不同颜色线条区分。

• 用户可在侧边栏切换查看单个支系或全部支系。

3.5.2 迁徙原因标注

• 在路径关键节点显示迁徙原因标签（如“战乱”、“经商”、“仕宦”、“垦荒”）。

• 标签以气泡形式出现在 POI 点上方，点击可查看详细文字记载。

3.5.3 数据缺失处理

• 若某个人物缺少地点信息，系统自动跳过，并在时间轴上标注“未知地点”。

• 支持管理员手动补充迁徙事件，补充后地图实时更新。

3.5.4 导出与分享

• 支持将迁徙动画录制为视频（MP4），用于线下播放。

• 支持生成静态迁徙路线图（PNG/JPG），可插入族谱印刷版。

3.5.5 移动端适配

• 手机端自动切换为竖屏模式，时间轴置于底部，地图占满屏幕。

• 图片缩略图改为单行横向滚动。

4. 数据架构

4.1 新增表

-- 迁徙事件表
CREATE TABLE migration_events (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  clan_id BIGINT NOT NULL,
  person_id BIGINT NULL,                    -- 关联人物（可选）
  branch_id BIGINT NULL,                    -- 关联支系（可选）
  from_location VARCHAR(200) NOT NULL,      -- 迁出地
  to_location VARCHAR(200) NOT NULL,        -- 迁入地
  from_lat DECIMAL(10,7),                   -- 迁出地纬度
  from_lng DECIMAL(10,7),                   -- 迁出地经度
  to_lat DECIMAL(10,7),                     -- 迁入地纬度
  to_lng DECIMAL(10,7),                     -- 迁入地经度
  event_year INT,                           -- 迁徙年份（模糊年份用负号表示，如 -1650 表示约1650年）
  reason VARCHAR(100),                      -- 迁徙原因（战乱/经商/仕宦/垦荒/其他）
  description TEXT,                         -- 详细描述
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 地点-图片关联表（用于迁徙地图展示）
CREATE TABLE migration_location_media (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  location_name VARCHAR(200) NOT NULL,      -- 地点名称
  media_id BIGINT NOT NULL,                 -- 关联的图片ID
  display_order INT DEFAULT 0,              -- 显示顺序
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 历史朝代数据表（预置数据）
CREATE TABLE historical_dynasties (
  id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,                -- 朝代名称
  start_year INT NOT NULL,
  end_year INT NOT NULL,
  geo_json_url VARCHAR(500),                -- 疆域GeoJSON数据地址
  description TEXT,
  color VARCHAR(20)                         -- 地图上显示的颜色
);


4.2 索引建议

• migration_events：索引 (clan_id, event_year) 用于按家族和时间排序。

• historical_dynasties：索引 (start_year, end_year) 用于时间匹配查询。

5.3 交互细节

5.3.1 地图初始化

• 默认视角为中国全境（若家族数据限于国内），或自动缩放至包含所有 POI 点的边界。

5.3.2 POI 点击

• 点击 POI 点弹出信息卡片，包含：

  • 地名、时间段、居住人数。

  • “查看图片”按钮（打开该地点关联的照片墙）。

  • “查看人物”按钮（跳转到在该地生活过的族人列表）。

5.3.3 路径动画

• 播放时，路径线段从起点向终点逐段点亮（类似水流效果），已走过的路径保持高亮。

5.3.4 图片光晕

• 使用 CSS filter: drop-shadow() 或 Canvas 绘制发光效果。

• 光晕颜色随朝代变化微调（如明朝偏红、清朝偏黄）。

6. 错误处理与边界条件

场景 处理方式

家族无任何地点数据 提示“暂无迁徙数据，请先在族谱中补充出生地和迁徙事件”

地点解析失败（地理编码无结果） 标记为“未知地点”，在地图上显示为灰色问号图标，允许管理员手动编辑经纬度

历史朝代数据缺失（如春秋战国） 使用纯色背景代替，标注“该时期疆域数据暂缺”

图片过多导致地图卡顿 限制同时显示的缩略图数量（最多 9 张），其余通过“查看更多”按钮加载

播放过程中用户切换支系 重置时间轴到该支系的起始时间，重新开始播放

7. 非功能性需求

• 性能：POI 点少于 100 个时，地图操作流畅无卡顿；图片缩略图懒加载。

• 数据精度：地理编码准确率 ≥ 95%（使用高德/腾讯地图 API）。

• 兼容性：支持 Chrome/Firefox/Edge 最新版本，移动端 Safari 和 Chrome。

• 可扩展性：朝代图层数据支持外部导入（管理员可上传自定义 GeoJSON）。

8. 验收标准（DoD）

1. 选择一个有 5 个以上地点数据的家族，迁徙地图自动生成 POI 点和路径线。
2. 点击“播放”，时间轴从最早时间点开始前进，地图视角跟随 POI 点移动，背景地图在明清交替时自动切换为对应朝代疆域图。
3. 播放过程中，POI 点周围的缩略图依次出现，每张图片带有金色光晕呼吸动画。
4. 用户可暂停播放，拖动时间轴跳转到任意时间点，地图和图片同步更新。
5. 多支系家族可切换查看不同支系的迁徙路径，路径颜色不同。
6. 导出功能可生成静态路线图（PNG）和动态视频（MP4）。

迁徙地图 · 技术选型与数据方案补充（Appendix A）

本附录作为《迁徙地图功能需求文档 v1.0》的补充，提供地图SDK选型分析、历史朝代数据获取方案以及具体实现建议。

A.1 地图SDK选型

A.1.1 选型结论

候选引擎 协议/成本 适用性 推荐度

Leaflet BSD-2-Clause，完全开源免费，底图可自托管 点、线、面、图片叠加、时间线驱动动画均能满足 ✅ 主选

使用 Leaflet 1.9.x 作为主地图引擎，搭配自托管或免费栅格瓦片底图，通过 GeoJSON 叠加朝代疆域图层。

A.1.2 底图方案（三层可切换）

层级 方案 成本 说明

现代底图（默认） CartoDB Voyager 淡色 免费（CC-BY） 视觉干净，不干扰图片光晕

卫星底图 ESRI World Imagery（公益免费） 免费（非商用需确认） 用于查看地貌

离线底图（可选） 自托管 OpenStreetMap 瓦片 一次性服务器成本 适用于内网或高稳定性场景
A.1.3 关键功能实现方式
功能 Leaflet 实现

POI 点 L.marker() + 自定义 HTML Icon（CSS 脉冲光圈）

迁徙路径线 L.polyline() + 渐变色（按时间深浅）

朝代多边形 L.geoJSON() 半透明填色 + 虚线边线

图片缩略图 绝对定位 HTML div（通过 L.control 或 L.tooltip 的 HTML 模式）

光晕呼吸动画 CSS @keyframes pulse + box-shadow 金色光晕

时间轴驱动 监听时间轴滑块变化，更新图层可见性和地图中心

A.2 历史朝代疆域数据方案

A.2.1 数据源推荐

数据源 格式 许可 精度 推荐用途

开源历朝 GeoJSON 合集（GitHub） GeoJSON 多种开源协议 朝代级粗略疆域面 v1.0 快速启动

CHGIS V6（哈佛×复旦） Shapefile CC0 / EULA 府/州/县级多边形 v1.1 深化

Natural Earth 古边界整理 Shapefile/GeoJSON Public Domain 国家级粗边界 辅助参考

A.2.2 实施路线

第一阶段（v1.0）：使用开源朝代级简化 GeoJSON，覆盖明、清、民国、现代四个时期（覆盖 95% 族谱迁徙数据）。每个 GeoJSON 文件大小约 200KB-1MB，可直接放在前端静态资源目录。

第二阶段（v1.1）：对接 CHGIS V6 数据，通过 ogr2ogr 工具将 Shapefile 转换为 GeoJSON，并简化坐标精度，替换为更精确的府县层级边界。

A.2.3 数据结构定义

interface DynastyBoundary {
  id: string;                    // "ming"
  name: string;                  // "明朝（1368-1644）"
  startYear: number;             // 1368
  endYear: number;               // 1644
  color: string;                 // "#8B1A1A" 朱砂红
  fillOpacity: number;           // 0.10
  geojsonUrl: string;            // "/static/geojson/dynasties/ming.geojson"
  labelPosition: [number, number]; // 朝代名标注位置 [lat, lng]
}


A.2.4 注意事项

• 历史边界与现代边界存在差异，朝代多边形仅作为氛围装饰层，不标注国界文字，不涉及敏感表述。

• 页面底部添加小字注释：“历代疆域图为示意性参考，以现代行政区划为准。”

A.3 性能与兼容性

• POI 点数量：Leaflet 在 DOM Marker 数量 < 1000 时流畅运行，家族迁徙地图通常不超过 200 个 POI，无需特殊优化。

• 图片缩略图：同时显示不超过 9 张，其余通过“查看更多”加载，避免 DOM 过多。

• 移动端适配：Leaflet 原生支持触摸事件，配合 map.locate() 可实现自动定位。

• 浏览器兼容：支持 Chrome/Firefox/Edge 最新版本，移动端 Safari/Chrome。

A.4 开发建议

1. 使用 @vue-leaflet/vue-leaflet 或自行封装 Leaflet Vue 组件。
2. 朝代 GeoJSON 数据预加载到 Pinia store，按时间轴切换时仅更新图层可见性，避免重复解析。
3. 图片光晕效果使用 CSS 而非 Canvas，保持与 Vue 响应式系统一致。
4. 迁徙路径动画使用 L.Polyline 的 setLatLngs 方法逐段绘制，配合 requestAnimationFrame 实现平滑流动。
