# 归宗合并功能开发完成报告

## 开发完成时间
2026-06-21

## 功能概述
基于《寻根路（xungenlu.cn）归宗合并功能需求文档 v1.0》实现的完整功能模块。

## 已完成的工作

### 一、数据库层变更

**修改文件**: `packages/db/prisma/schema.prisma`

1. **扩展 ApplicationStatus 枚举**
   - 新增 `MERGED` - 归宗合并完成
   - 新增 `REVERTED` - 合并已回滚

2. **扩展 NotificationType 枚举**
   - 新增 `MERGE_COMPLETED` - 归宗合并完成通知
   - 新增 `MERGE_ROLLBACK` - 归宗合并回滚通知

3. **扩展 DataSnapshot 模型**
   - `merge_application_id` - 关联的合并申请
   - `applicant_clan_id` - 申请方家族ID（回滚用）
   - `merge_target_id` - 挂载点人物ID
   - `branch_root_id` - 申请方始祖ID
   - `operation_type` - 操作类型标识
   - `is_reverted` - 是否已回滚

4. **扩展 FamilyUnit 模型**
   - `union_type` - 标记家庭单元来源（normal/merger）

5. **数据库同步**
   - 使用 `prisma db push` 成功同步到 Neon 数据库
   - 迁移脚本已保存在 `docs/database/migration_add_merge_fields.sql`

### 二、后端服务开发

**新增文件**:
- `apps/server/src/admin/merge/merge.service.ts` - 合并服务核心逻辑
- `apps/server/src/admin/merge/merge-wizard.controller.ts` - 合并向导控制器

**修改文件**:
- `apps/server/src/admin/admin.module.ts` - 注册新服务
- `apps/server/src/common/notification.service.ts` - 添加合并相关通知方法

#### MergeService 核心功能

1. **挂载点验证**
   - 检查是否为已故人员
   - 检查是否属于申请方支系（防止循环）
   - 检查世代偏差警告

2. **世代对齐预览**
   - 计算申请方成员的世代偏移
   - 检测父子关系冲突

3. **合并执行**
   - 创建数据快照（主家族 + 申请方家族）
   - 创建合并家庭单元
   - 更新成员 clan_id
   - 闭包表重算

4. **合并回滚**
   - 恢复快照数据
   - 更新申请状态
   - 标记快照已回滚

#### 向导 API 端点

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/admin/merge/wizard/:appId | 获取向导初始化数据 |
| GET | /api/admin/merge/clans/:clanId/tree | 获取族谱树结构 |
| GET | /api/admin/merge/persons/:personId/ancestors | 获取祖先链 |
| POST | /api/admin/merge/validate-anchor | 验证挂载点合法性 |
| POST | /api/admin/merge/preview-alignment | 预览世代对齐 |
| POST | /api/admin/merge/execute | 执行合并 |
| GET | /api/admin/merge/merge-snapshots | 获取可回滚快照 |
| POST | /api/admin/merge/rollback/:snapshotId | 回滚合并 |

### 三、前端页面开发

**新增文件**:
- `apps/web/src/components/TreeSelect.vue` - 族谱树选择器组件
- `apps/web/src/views/admin/MergeWizardPage.vue` - 合并向导页面

**修改文件**:
- `apps/web/src/views/admin/MergeApplicationsPage.vue` - 增强申请列表
- `apps/web/src/router/index.ts` - 添加向导路由

#### TreeSelect 组件功能

- 双树展示（主家族 + 申请方支系）
- 节点搜索
- 缩放控制（50%-150%）
- 已故/在世节点样式区分
- 单选模式（选择挂载点）

#### MergeWizardPage 向导页面

四步骤流程：
1. **挂载点选择** - 选择主家族中的已故人员
2. **世代确认** - 预览并微调世代对齐
3. **执行合并** - 确认并执行事务
4. **完成** - 展示合并结果

#### MergeApplicationsPage 增强

- 新增状态标签：`MERGED`（已合并）、`REVERTED`（已回滚）
- "初审通过"按钮 - 进入待合并状态
- "执行合并"按钮 - 跳转到合并向导
- 回滚快照对话框增强

#### 路由配置

{

}
