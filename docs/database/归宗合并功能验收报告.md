## 四、核心功能验证

### 4.1 挂载点验证逻辑 ✅

```
- 检查挂载点是否存在且属于主家族
- 检查是否为已故人员（必须）
- 检查是否属于申请方支系（防止循环）
- 检查世代偏差警告（>2代警告）
```

### 4.2 世代对齐预览 ✅

```
- 计算申请方成员的世代偏移
- 检测父子关系冲突（年龄校验）
- 支持微调世代偏移（±1）
```

### 4.3 合并执行事务 ✅

```
1. 创建完整数据快照（主家族+申请方家族）
2. 创建合并家庭单元（union_type='merger'）
3. 将申请方始祖添加为挂载点子女
4. 更新所有成员 clan_id
5. 重算闭包表（闭包表重算算法）
6. 更新申请状态为 MERGED
7. 发送合并完成通知
```

### 4.4 回滚机制 ✅

```
1. 检查快照是否已回滚
2. 检查是否过期（24小时）
3. 恢复主家族数据
4. 恢复申请方家族数据
5. 恢复家庭单元和子女关系
6. 恢复闭包表
7. 更新申请状态为 REVERTED
8. 标记快照为已回滚
```
