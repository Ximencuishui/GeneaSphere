用户中心 · 家庭图册功能需求 v1.0

本文档是对《用户中心页面需求文档 v1.0》和《印刷模块》的补充，新增“家庭图册”功能，支持从某位前辈开始，生成其后代若干代的家庭成员信息图文，以电子杂志形式预览，并可下单印刷。

1. 功能概述

1.1 背景

家族中常有需求：为某位长辈（如爷爷）制作一本“家庭纪念册”，收录他/她往后几代所有家庭成员的信息（姓名、照片、简介、关系）。这类图册常用于祝寿、祭祖、家族聚会等场景。现有印刷模块只能印刷族谱树，无法生成精美的图文杂志。

1.2 目标

• 用户选择一位前辈（如爷爷），指定向后延展的代数（如到孙子辈）。

• 系统自动收集该前辈所有后代（包括配偶）的基本信息和照片。

• 生成一本电子杂志风格的图文册，可在线预览，并支持一键下单印刷。

• 用户可自定义分类（如按房支、按家庭分组），选择包含的信息字段。

1.3 用户故事

• 作为孙子：我想为爷爷90大寿制作一本家庭图册，收录爷爷、爸爸、叔叔、我和堂兄妹们的照片和简介。

• 作为家族管理员：我想为开基祖制作一本“后世子孙图册”，展示500年来家族的发展。

2. 功能详情

2.1 触发入口

• 用户中心 → 工具箱：新增“家庭图册”卡片。

• 族谱树：右键点击人物节点，菜单中增加“生成家庭图册”。

2.2 参数设置

参数 说明 默认值

起始人物 选择一位前辈（必选） 当前用户的上3代祖先（如爷爷）

向后代数 从起始人物向后延展的代数（1-10） 3

包含配偶 是否包含每代人的配偶 是

分类方式 按房支 / 按家庭（每对夫妻为一组） / 不分类（按世代） 按家庭

信息字段 勾选要展示的信息：姓名、生卒年、照片、简介、职业、住址等 默认全选

封面风格 提供5种模板（喜庆红、典雅金、清新绿、水墨风、现代简约） 喜庆红

标题 自定义图册标题，如“张氏家族·三代同堂” 自动生成

2.3 内容生成逻辑

• 系统从起始人物开始，沿血缘向下查找指定代数的所有后代（包括配偶）。

• 按分类方式组织内容：

  • 按家庭：每对夫妻及其子女为一组，每组一页或两页。

  • 按房支：按起始人物的子女分支分组（如长房、二房）。

  • 按世代：按辈分分组（第一代、第二代...）。

• 每个人物展示所选信息字段，照片优先从 media_archive 和 user_photos 中提取，若无则显示占位头像。

• 自动生成目录页和前言（可编辑）。

2.4 电子杂志预览

• 使用翻页电子杂志组件（如 turn.js 或类似），模拟书本翻页效果。

• 支持全屏预览、缩放、跳转到指定页码。

• 用户可在线编辑文字内容（如修改简介、添加祝福语）。

2.5 印刷下单

• 预览满意后，点击“下单印刷”进入印刷模块。

• 自动带入图册的页数、规格（默认A4精装，可改）。

• 支持一本起订，价格按页数计算（参考印刷模块定价）。

3. 数据架构

3.1 新增表

-- 家庭图册项目表
CREATE TABLE family_book_projects (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  start_person_id BIGINT NOT NULL,
  generations INT DEFAULT 3,
  include_spouse BOOLEAN DEFAULT TRUE,
  grouping ENUM('family', 'branch', 'generation') DEFAULT 'family',
  selected_fields JSON,                      -- ["name","birth","photo","bio","occupation"]
  cover_template VARCHAR(50) DEFAULT 'red',
  title VARCHAR(100),
  page_count INT,
  preview_url VARCHAR(500),
  status ENUM('draft', 'preview', 'ordered') DEFAULT 'draft',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 图册页面内容表（缓存生成的内容，便于编辑和重新生成）
CREATE TABLE book_pages (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  project_id BIGINT NOT NULL,
  page_number INT NOT NULL,
  page_type ENUM('cover', 'toc', 'section', 'person', 'family_group') DEFAULT 'person',
  content JSON NOT NULL,                     -- 包含标题、人物列表、照片URL等
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


3.2 与现有模块的关系

• 复用印刷模块的订单系统和模板引擎。

• 人物数据来源于 persons 表。

• 照片来源于 media_archive 和 user_photos。

4.2 预览页面（@/pages/Tools/FamilyBookPreview.vue）

• 全屏翻页电子杂志。

• 底部工具栏：上一页/下一页、跳转页、全屏、编辑文字、下单印刷。

4.3 编辑功能

• 点击页面中的文字区域可弹出编辑框，修改后保存。

• 支持替换照片（从个人相册或家族影像库中选择）。

• 编辑后重新生成预览（保留修改）。

5. 验收标准

1. 选择爷爷作为起始人物，向后3代，生成的图册包含爷爷、父母辈、自己辈、子女辈的所有家庭成员（含配偶）。
2. 按家庭分组时，每对夫妻及其子女显示在同一页或相邻页。
3. 电子杂志翻页流畅，文字和图片清晰。
4. 点击“下单印刷”跳转到印刷模块，页数和规格正确。
5. 用户可在线编辑文字内容，编辑后预览更新。

6. 后续迭代

• v1.1：支持添加自定义章节（如“家族大事记”、“童年回忆”）。

• v1.2：AI自动生成每人的简介（基于族谱数据和AI）。

• v1.3：支持多人协作编辑（如家族成员共同修改图册内容）。
