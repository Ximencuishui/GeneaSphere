寻根路（xungenlu.cn）Admin 管理后台需求文档 v1.0

文档版本：v1.0  

对应系统版本：寻根路 GeneaSphere Platform v1.0  

面向对象：开发团队、测试团队、产品经理  

前置依赖：用户系统、家族系统、支付系统、COS 存储、短信服务  

1. 产品概述

1.1 背景

寻根路平台包含两类管理角色：家族管理员（管理单个家族内部事务）和平台管理员（管理全平台运营）。两者需要统一的后台入口，但功能和权限严格分离。本需求文档定义 Admin 管理后台的整体架构、功能模块和交互规范。

1.2 目标

• 为家族管理员提供家族内部管理工具（成员、内容、设置、印刷、短信）。

• 为平台管理员提供全局管控工具（家族审核、用户封禁、内容审查、订单处理、系统配置）。

• 确保多租户隔离：家族管理员只能操作本家族数据，平台管理员可操作所有家族数据。

2. 角色与权限

角色 所属层级 权限范围

家族超级管理员（Owner） 家族 家族内所有权限，可转让所有权

家族管理员（Admin） 家族 内容审核、寻亲处理、系统设置、成员管理（不可删除 Owner）

家族编辑者（Editor） 家族 增删改人员信息、上传照片、修改生平

家族观察员（Viewer） 家族 仅查看

平台超级管理员 平台 全部权限，不可被限制

平台运营管理员 平台 内容审核、订单管理、数据查看，不可修改系统配置

平台财务管理员 平台 订单退款、充值对账，不可查看用户隐私

平台审计员 平台 仅查看操作日志和数据统计

3. 功能模块

3.1 登录与安全

• 独立登录页（域名：admin.xungenlu.cn）。

• 支持手机号+密码登录，平台管理员额外支持双因素认证（v1.1）。

• 登录后根据角色跳转至对应的控制台首页。

• 密码错误超过 5 次锁定 30 分钟。

3.2 控制台首页

• 家族管理员：显示本家族成员数、照片数、存储用量、待审核内容数、短信余额、近期动态。

• 平台管理员：显示全平台家族数、用户数、照片数、存储总量、今日新增、待处理事项（审核、退款）、收入概览。

3.3 家族管理（仅平台管理员可见）

3.3.1 家族列表

• 字段：家族 ID、名称、管理员、成员数、注册时间、状态（正常/冻结/待审核）。

• 支持搜索、筛选、导出。

• 操作：查看详情、冻结/解冻、删除（二次确认）、数据导出。

3.3.2 家族审核

• 新注册家族进入待审核列表。

• 审核项：名称合规性、管理员身份真实性（可要求上传证件）。

• 操作：通过 / 驳回（填写理由）。

3.4 用户管理（平台管理员）

3.4.1 用户列表

• 字段：用户 ID、昵称、手机号（脱敏）、所属家族、注册时间、状态（正常/封禁）。

• 支持搜索、筛选。

• 操作：查看详情、封禁/解封、重置密码、删除账号。

3.4.2 用户详情

• 显示个人信息、关联家族、操作日志（最近 20 条）。

3.5 内容审核（家族管理员 + 平台管理员）

3.5.1 影像审核

• 家族管理员：审核本家族成员上传的照片/视频。

• 平台管理员：审核全平台被举报或 AI 标记的敏感内容。

• 操作：通过 / 删除 / 忽略。

3.5.2 寻亲帖审核

• 家族管理员：审核本家族寻亲帖。

• 平台管理员：审核全平台寻亲帖。

• 操作：通过 / 下架。

3.5.3 举报管理

• 平台管理员处理用户举报，确认违规则删除内容并警告举报人。

3.6 寻亲与归宗管理（家族管理员）

3.6.1 认亲申请列表

• 展示申请加入本家族的请求。

• 字段：申请人、祖籍地、字辈、匹配度、状态。

• 操作：查看详情、通过（执行归宗合并）、拒绝。

3.6.2 归宗合并工具

• 选择挂载点、确认世代偏移、执行合并（含快照和回滚能力）。

• 详细流程见《归宗合并功能需求文档》。

3.7 系统设置（家族管理员 + 平台管理员）

3.7.1 隐私配置（家族管理员）

• 游客可见范围（代数、在世人员隐藏等）。

• 亲属验证登录配置（问题类型、容错次数）。

3.7.2 字辈管理（家族管理员）

• 维护本家族字辈序列，新增人员时自动校验。

3.7.3 全局配置（平台管理员）

• 短信单价、AI 工具定价、免费额度、印刷基准价。

• 注册审核开关、短信功能开关、AI 工具开关。

3.8 印刷订单管理（家族管理员 + 平台管理员）

3.8.1 家族管理员

• 查看本家族订单，跟踪物流，再次购买。

3.8.2 平台管理员

• 查看全平台订单，标记发货，处理退款。

• 对账导出。

3.9 短信通知（家族管理员）

• 选择收件人（按房支、世代、地区等），编辑内容，立即或定时发送。

• 采用余额钱包预付制，详见《家族短信通知功能需求文档》。

3.10 日志审计（家族管理员 + 平台管理员）

• 家族管理员：查看本家族关键操作日志。

• 平台管理员：查看全平台管理员操作日志。

• 日志不可删除，保留至少 1 年。

3.11 数据统计（平台管理员）

• 日报/周报/月报自动生成 PDF。

• 家族排行（成员数、照片数、消费金额）。

• 工具使用排行。

5. 数据接口

所有接口需校验管理员 Token 和角色权限。主要接口分类：

模块 接口示例 权限

控制台 GET /admin/dashboard 所有管理员

家族管理 GET /admin/families 平台管理员

用户管理 PATCH /admin/users/:id/ban 平台管理员

内容审核 POST /admin/reviews/media/:id/approve 家族/平台管理员

寻亲管理 POST /admin/searches/applications/:id/merge 家族管理员

系统设置 PUT /admin/settings/privacy 家族管理员

订单管理 POST /admin/orders/:id/refund 平台管理员

短信 POST /admin/sms/send 家族管理员

日志 GET /admin/logs 所有管理员

6. 非功能性需求

• 性能：列表页加载 < 2s，操作响应 < 500ms。

• 安全：所有接口校验角色权限；敏感操作二次确认；日志不可篡改。

• 多租户隔离：家族管理员接口强制校验 clan_id 与当前管理员所属家族一致。

• 可用性：99.5% 正常运行时间。

7. 验收标准

1. 家族管理员登录后只能看到本家族数据，无法操作其他家族。
2. 平台管理员可查看所有家族，可冻结/解冻家族。
3. 内容审核通过后，前端用户可见；删除后不可见。
4. 短信发送功能正常扣费，余额不足时提示充值。
5. 所有关键操作记录在日志中，且不可删除。
