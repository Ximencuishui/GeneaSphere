1. 项目愿景与使命

1.1 愿景

打造中国首个集数字化修谱、智能化寻亲、聚落化记忆于一体的文化传承平台。不仅记录血脉的延续，更留存家族的温度与乡土的根基。

1.2 使命

解决传统修谱“门槛高、成本高、易失传”的痛点，通过技术手段，让每一个中国家庭都能轻松拥有一部“活的族谱”，让年轻一代看得见历史，让年长一代留得住回忆。

1.3 核心价值主张

•   于个人：找到我是谁，我从哪里来，我的根在何处。

•   于家族：打破地域限制，凝聚族人，续写家史。

•   于社会：数字化保存村落记忆，助力文化自信与乡村振兴。

2. 市场痛点分析

1.  老谱断层：现存老谱多为文言/繁体，破损严重，且大多止步于建国前，后续族人信息缺失。
2.  修谱门槛高：传统修谱耗资巨大、周期长（数年），需专人编纂，普通家族难以承担。
3.  数据孤岛：分散各地的族人信息无法汇聚，微信群里的聊天记录无法转化为结构化的族谱数据。
4.  记忆流失：老一辈口述历史无人记录，老照片散落各地，随着岁月流逝面临永久消失的风险。
5.  寻亲困难：因历史迁徙、战乱，大量支系失联，仅凭口耳相传难以精准对接。

3. 产品核心功能详解

3.1 智能数据录入与建树（地基）

•   多源导入：

    ◦   Excel导入：支持标准化模板一键导入，自动识别父子、配偶关系。

    ◦   PDF/图片导入：针对老谱扫描件，利用OCR技术提取文字，结合人工校对后台，将非结构化数据转化为结构化族谱。

    ◦   手写/老照片识别：识别墓碑、老证件上的关键信息，辅助完善人物详情。

•   自动建树：基于导入数据，自动生成可视化家族树，支持拖拽调整节点位置。

•   中式关系适配：完美支持“字辈”、“过继”、“入赘”、“兼祧”等中国特色宗法关系逻辑。

3.2 可视化交互与多端适配（展示）

•   Xmind式思维导图：支持树状图、思维导图、时间轴等多种视图切换。用户可聚焦于任意个体，展开其上下五代关系网。

•   高性能渲染：采用Canvas技术，流畅支持数万节点渲染，告别卡顿。

•   响应式设计：PC端侧重编辑与管理，移动端侧重浏览与分享。

3.3 个人详情“博客页”（灵魂）

•   每位族人拥有独立主页，类似一篇传记。

•   生平事迹：支持富文本编辑，记录生平大事、职业成就、趣闻轶事。

•   影像馆：聚合与该人物相关的所有照片、视频、录音（口述历史）。

•   血缘路径：清晰展示“你是某某的第几代孙”。

3.4 寻亲与支系合并（增长引擎）

•   发布寻亲帖：用户可发布包含“祖籍地、字辈、关键祖先名”的寻亲信息。

•   智能匹配：系统基于“字辈相似度”和“地名重合度”自动推荐潜在关联家族。

•   归宗仪式感：寻亲成功后，提供可视化的“支系挂载”引导，并生成“归宗纪念页”，支持一键更新族谱印刷版。

3.5 时光长廊与聚落记忆（情感壁垒）

•   家族时光轴：按时间线展示家族重大事件与老照片。

•   聚落影像互联（核心特色）：

    ◦   建立“村寨/聚落”维度，打破单一家族界限。

    ◦   同频推荐：系统自动识别拍摄于同一年代、同一村寨（或附近村寨）的照片，即使属于不同家族，也会在“聚落时光长廊”中共同展示。

    ◦   价值：还原真实的乡土生活场景（如全村春耕、集体婚礼），让后人看到先辈不仅仅是血缘个体，更是社会共同体的一员。

3.6 一键印刷与出版（商业闭环）

•   模板化排版：提供仿古宣纸、现代精装、红底金字等多种PDF模板。

•   所见即所得：在线预览印刷效果，支持分卷（如：长房卷、二房卷）。

•   一本起订：对接数码快印供应链，支持小批量定制，大幅降低门槛。

•   增值服务：支持将“时光长廊”精选照片植入族谱，或制作成纪念台历、挂历。

3.7 安全与权限体系（信任基石）

•   家族自治：每个家族设立管理员，负责审核数据、管理成员。

•   独创“亲属验证”登录：新成员加入需回答“您的祖父叫什么名字”等问题，确保数据仅对族人开放。

•   隐私保护：

    ◦   在世之人：默认对游客隐藏详细信息，仅显示称谓。

    ◦   游客范围控制：管理员可设置游客仅能查看五代以外或已故人员信息。

•   数据主权：用户数据可随时导出，永不锁死。

4. 技术架构概览

4.1 总体架构

•   前端：Vue 3 + Vite + AntV G6（可视化图谱）。

•   后端：Java Spring Boot 或 Node.js NestJS（构建稳健的业务逻辑）。

•   数据库：

    ◦   MySQL/PostgreSQL：存储基础信息、用户数据、权限配置。

    ◦   闭包表（Closure Table）：专门用于存储和查询复杂的血缘层级关系，支持高效查询祖先/后代。

•   存储：对象存储（OSS/S3）用于存放海量的图片、视频和PDF文件。

•   部署：Docker 容器化部署，便于弹性伸缩。

4.2 关键技术点

•   闭包表算法：解决传统递归查询慢的问题，实现毫秒级族谱树展开。

•   OCR与NLP：用于解析老谱PDF和图片中的非结构化文本。

•   图像感知哈希（pHash）：用于实现“同村同年代照片推荐”，识别相似场景的图片内容。

•   GeoHash：用于地理位置索引，快速查找“附近村寨”。

5. 商业模式

1.  基础服务免费：建树、基础浏览、Excel导入永久免费，以此获取海量用户。
2.  增值服务（VIP）：
    ◦   云存储扩容：超出免费额度的照片/视频存储。

    ◦   高级模板：精美印刷模板、视频纪念册制作。

3.  核心收入：印刷服务：族谱书籍、纪念画册、台历的按需印刷（POD）。
4.  B端服务：为大型宗祠、姓氏文化研究会提供定制化SaaS解决方案。

6. 运营推广策略

1.  内容营销：撰写“如何修谱”、“字辈文化”、“寻根故事”等高质量文章，通过公众号、头条号分发。
2.  种子用户：寻找正在修谱的家族、姓氏文化爱好者、退休教师作为首批体验官。
3.  线下合作：与印刷厂、家谱馆、宗祠建立合作，将其线下客户引流至线上平台。
4.  寻亲驱动：利用“寻亲”功能的高粘性，形成自传播。一旦寻亲成功，该支系用户将迅速转化为平台活跃用户。
5.  村志联动：与地方村委合作，将“聚落影像”功能应用于“村志编撰”，提升平台公信力。

7. 项目里程碑（Roadmap）

•   第一阶段：MVP（第1-3个月）

    ◦   完成核心建树、Excel导入、基础树形展示、PDF导出。

    ◦   上线基础印刷对接流程（人工介入）。

    ◦   目标：跑通“数据录入 -> 生成文件 -> 印刷成书”的商业闭环。

•   第二阶段：情感连接（第4-6个月）

    ◦   上线“时光长廊”与“个人博客页”。

    ◦   上线“寻亲广场”与“亲属验证登录”。

    ◦   目标：提升用户粘性，验证寻亲功能的有效性。

•   第三阶段：壁垒构建（第7-12个月）

    ◦   上线“聚落影像互联”功能。

    ◦   实现图片/PDF的智能识别与校对。

    ◦   对接自动化印刷API，实现全流程无人化。

    ◦   目标：确立行业技术壁垒，扩大市场份额。

8. 风险与应对

风险类别 潜在风险 应对策略

数据隐私 在世族人信息泄露 严格的权限分级；默认脱敏；加密存储；法律声明。

数据质量 老谱识别错误、用户乱填 建立“管理员审核”机制；引入版本回溯功能；OCR+人工双重校对。

市场竞争 现有修谱公司转型 发挥互联网优势（低成本、高效率、跨地域）；强化“影像”与“寻亲”差异化功能。

技术成本 OCR/AI 识别费用高 前期采用第三方API；后期积累数据训练自有轻量模型；优化算法降低调用频次。
