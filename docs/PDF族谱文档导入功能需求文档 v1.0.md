文档版本：v1.0

对应系统版本：GeneaSphere Platform v1.0

前置依赖：现有Excel导入模块、文件上传系统、族谱数据架构

## 1. 功能概述

### 1.1 背景

当前系统已支持Excel格式（.xlsx）的族谱数据导入，但许多家族持有的历史族谱以PDF格式保存（如扫描件、电子版家谱等）。为满足用户从PDF族谱文档中提取和导入人员数据的需求，需新增PDF文档导入功能。

### 1.2 功能目标

- 支持用户上传PDF格式的族谱文档
- 自动解析PDF内容，提取人员信息（姓名、性别、出生日期、逝世日期、辈分等）
- 将提取的数据转换为系统标准格式并导入数据库
- 提供解析结果预览和人工校对功能
- 支持PDF文本版和扫描图片版的智能识别

### 1.3 适用场景

| 场景 | 说明 |
|------|------|
| 历史族谱数字化 | 将纸质族谱扫描件（PDF）导入系统 |
| 电子族谱迁移 | 从其他系统导出的PDF族谱文件导入 |
| 家族文档归档 | 将家族PDF文档中的人员信息整合到系统中 |

### 1.4 功能边界

- **支持**：结构化/半结构化PDF文本、PDF扫描件（OCR识别）
- **不支持**：纯图片PDF（无文字层）、加密PDF、手写体PDF（首期）

## 3. 功能列表

### 3.1 PDF文件上传

**功能描述**：用户上传PDF族谱文档

**需求要点**：
- 支持拖拽上传和点击上传
- 文件大小限制：最大50MB
- 文件格式：仅支持.pdf
- 显示上传进度条
- 上传前进行基础校验（格式、大小）

### 3.2 PDF内容解析

**功能描述**：系统自动解析PDF内容，提取人员数据

**解析模式**：

#### 3.2.1 文本PDF解析
- 直接提取PDF中的文本内容
- 使用正则表达式和NLP技术识别人员信息
- 识别常见族谱格式（如世系表、家传体等）

#### 3.2.2 扫描件PDF解析（OCR）
- 调用OCR服务识别图片中的文字
- 支持简体中文、繁体中文识别
- 识别准确率目标：≥85%

**提取字段**：

| 字段 | 说明 | 必填 |
|------|------|------|
| full_name | 姓名 | 是 |
| gender | 性别（男/女） | 否（可从上下文推断） |
| generation | 辈分/世代 | 否 |
| birth_date | 出生日期 | 否 |
| death_date | 逝世日期 | 否 |
| is_living | 是否在世 | 否 |
| parent_name | 父亲/母亲姓名 | 否 |
| spouse_name | 配偶姓名 | 否 |
| biography | 生平简介 | 否 |
| burial_place | 葬地 | 否 |
| notes | 备注 | 否 |

### 3.3 智能映射与推断

**功能描述**：系统自动推断数据关系和缺失字段

**智能能力**：
- **辈分推断**：根据"第X世"、"X代"等关键词自动标记辈分
- **性别推断**：从称谓（公、氏、夫人等）推断性别
- **父子关系识别**：识别"长子"、"次子"、"之女"等关系描述
- **日期标准化**：将农历、年号等转换为公历（可选）
- **在世状态推断**：根据是否有逝世日期推断

### 3.4 解析结果预览与校对

**功能描述**：用户查看解析结果并进行人工校对

**界面要求**：
- 表格形式展示解析结果
- 高亮显示低置信度字段（置信度<70%）
- 支持手动编辑每条记录
- 支持批量操作（批量修改、批量删除）
- 显示原始PDF片段（便于对照）

**预览字段**：
- 行号
- 姓名
- 性别
- 辈分
- 出生日期
- 逝世日期
- 父/母姓名
- 置信度评分
- 原始文本片段

### 3.5 导入确认与执行

**功能描述**：用户确认解析结果后执行导入

**流程**：
1. 显示导入统计（成功数、失败数、待确认数）
2. 显示冲突检测（与现有人员重名等）
3. 用户选择冲突处理策略（跳过、合并、新增）
4. 执行导入
5. 显示导入结果报告

### 3.6 导入历史记录

**功能描述**：记录所有PDF导入操作

**记录内容**：
- 导入时间
- 文件名
- 文件大小
- 解析模式（文本/OCR）
- 成功条数
- 失败条数
- 操作人
- 导入状态（成功/部分成功/失败）

## 4. 技术架构

### 4.1 后端技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| PDF解析 | pdf-parse / pdf2json | 提取PDF文本内容 |
| OCR引擎 | Tesseract.js 或 百度OCR API | 扫描件文字识别 |
| NLP处理 | 自定义规则引擎 + 正则 | 人员信息提取 |
| 文件存储 | 本地存储 / 云存储 | 临时存储上传的PDF |

### 4.2 前端技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| PDF预览 | vue-pdf-embed | 在线预览PDF原文 |
| 表格编辑 | Element Plus Table | 数据校对编辑 |
| 上传组件 | Element Plus Upload | 文件上传 |

### 4.3 API设计

```
POST /api/import/pdf
- 上传PDF文件
- 返回：解析任务ID

GET /api/import/pdf/:taskId/status
- 查询解析任务状态
- 返回：解析进度、置信度统计

GET /api/import/pdf/:taskId/preview
- 获取解析结果预览
- 返回：人员数据列表、原始文本片段

PUT /api/import/pdf/:taskId/correct
- 提交校对修改
- Body：修改后的人员数据

POST /api/import/pdf/:taskId/execute
- 执行导入
- 返回：导入结果统计

GET /api/import/pdf/history
- 获取导入历史记录
```

### 4.4 数据流程

```
用户上传PDF 
  → 文件校验（格式、大小）
  → PDF类型检测（文本/扫描）
  → [文本PDF] 直接提取文本
  → [扫描PDF] OCR识别文字
  → NLP解析人员信息
  → 生成预览数据
  → 用户校对
  → 冲突检测
  → 执行导入
  → 写入数据库
  → 生成报告
```

## 5. 数据库设计

### 5.1 PDF导入记录表

```sql
CREATE TABLE pdf_import_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id VARCHAR(36) NOT NULL,
  clan_id BIGINT NOT NULL,
  file_name VARCHAR(500) NOT NULL,
  file_size BIGINT NOT NULL,
  file_path VARCHAR(1000) NOT NULL,
  parse_mode ENUM('text', 'ocr') NOT NULL,
  total_records INT DEFAULT 0,
  success_records INT DEFAULT 0,
  failed_records INT DEFAULT 0,
  status ENUM('pending', 'parsing', 'preview', 'correcting', 'importing', 'completed', 'failed') DEFAULT 'pending',
  error_message TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  completed_at TIMESTAMP NULL,
  INDEX idx_user_id (user_id),
  INDEX idx_clan_id (clan_id),
  INDEX idx_status (status),
  INDEX idx_created_at (created_at)
);
```

### 5.2 PDF解析临时数据表

```sql
CREATE TABLE pdf_parse_temp (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  import_log_id BIGINT NOT NULL,
  row_number INT NOT NULL,
  full_name VARCHAR(200) NOT NULL,
  gender ENUM('M', 'F', 'UNKNOWN') DEFAULT 'UNKNOWN',
  generation INT,
  birth_date DATE,
  death_date DATE,
  is_living BOOLEAN DEFAULT TRUE,
  parent_name VARCHAR(200),
  spouse_name VARCHAR(200),
  biography TEXT,
  burial_place VARCHAR(500),
  notes TEXT,
  confidence_score DECIMAL(5,2),
  original_text TEXT,
  is_corrected BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_import_log_id (import_log_id),
  INDEX idx_confidence (confidence_score),
  FOREIGN KEY (import_log_id) REFERENCES pdf_import_logs(id) ON DELETE CASCADE
);
```

## 7. 错误处理与边界条件

| 场景 | 处理策略 |
|------|---------|
| PDF文件损坏 | 提示"文件无法解析，请检查文件完整性" |
| PDF加密 | 提示"不支持加密PDF，请先解密" |
| 文件大小超限 | 提示"文件大小超过50MB限制" |
| OCR识别失败 | 提示"扫描件识别失败，请确保图片清晰" |
| 解析出0条记录 | 提示"未能识别到人员信息，请检查文档格式" |
| 低置信度记录过多 | 警告"低置信度记录占比超过30%，建议人工校对" |
| 导入冲突 | 提供冲突解决选项（跳过/合并/覆盖） |
| 内存不足 | 分块处理大文件，提示"文件过大，正在分段处理" |

## 8. 非功能性需求

### 8.1 性能要求

- PDF解析速度：≤30秒（50MB以内文件）
- OCR识别速度：≤60秒（50MB以内文件）
- 并发支持：同时支持10个PDF解析任务
- 导入执行时间：≤10秒（1000条记录）

### 8.2 准确性要求

- 文本PDF解析准确率：≥95%
- 扫描件OCR识别准确率：≥85%
- 人员信息提取准确率：≥90%
- 父子关系识别准确率：≥80%

### 8.3 安全要求

- 上传文件病毒扫描
- 文件存储加密
- 用户权限校验
- 操作日志记录
- 临时数据自动清理（导入完成后7天）

### 8.4 兼容性要求

- PDF版本：1.0 - 1.7
- 编码支持：UTF-8、GB2312、GBK
- 浏览器支持：Chrome、Firefox、Safari、Edge（最新2个版本）
