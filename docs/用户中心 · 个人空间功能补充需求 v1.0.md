用户中心 · 个人空间功能补充需求 v1.0

本附录是对《用户中心页面需求文档 v1.0》的补充，新增“个人空间”模块，包含相册和留言板功能，类似 QQ 空间的轻量社交体验，但紧密围绕家族场景设计。

1. 功能概述

1.1 背景

用户中心目前偏重“数据管理”属性（我的照片、我的订单、我的工具等），缺乏一个展示自我、表达情感、与族人互动的空间。借鉴 QQ 空间的“相册”和“留言板”概念，但针对家族场景进行改造——相册强调时空标注和隐私分级，留言板强调轻量交流和家族纽带。

1.2 目标

• 为用户提供一个个人展示空间，可创建相册并上传照片，每张照片必须标注地点、年份、时间。

• 相册和单张照片均支持细粒度隐私设置（全族公开 / 上下几代公开 / 平台公开 / 同地点用户公开）。

• 提供留言板功能，类似微博客，每条留言限制 200 字，支持文字和图片，隐私设置与相册一致。

• 留言板内容可被其他家族成员查看、评论（评论功能 v1.1 实现）。

2. 角色与权限

角色 权限

本人 完全控制自己的相册和留言板（创建、编辑、删除、设置隐私）

同家族成员 根据隐私设置查看相册和留言板内容

跨家族用户 仅当设置为“平台公开”或“同地点用户公开”时可查看

游客 不可查看任何个人空间内容

3. 功能列表

3.1 个人相册

3.1.1 相册管理

创建相册
• 入口：用户中心 → 个人空间 → 相册 → “创建相册”按钮。

• 表单字段：

  • 相册名称（必填，最多 20 字）

  • 相册封面（选填，从已上传照片中选择或新上传）

  • 相册描述（选填，最多 100 字）

  • 默认隐私设置（新建照片时继承此设置，可单张修改）

编辑/删除相册
• 可修改名称、描述、封面、默认隐私设置。

• 删除相册时，相册内所有照片移至“未分类”相册（不删除照片本身）。

相册列表
• 以卡片网格展示，每个卡片显示封面、名称、照片数、最后更新时间。

• 支持按创建时间、更新时间排序。

3.1.2 照片管理

上传照片到相册
• 单次最多上传 9 张。

• 每张照片必须填写以下元数据（必填，否则无法上传）：

  • 地点：文本输入 + 自动补全（基于已有 POI 库）

  • 年份：数字输入（1900-当前年份）

  • 时间：日期选择器（精确到日，可选精确到月/年）

• 可选字段：人物标签（关联家族成员）、描述（最多 100 字）。

存储限制
• 每个用户的个人相册总容量上限 200MB。

• 在相册列表顶部显示已用容量/总容量进度条。

• 超出后无法上传新照片，提示“个人相册容量已满，请删除旧照片或联系管理员扩容”。

照片操作
• 查看大图：点击照片进入大图预览模式，显示完整元数据。

• 编辑：修改地点、年份、时间、描述、隐私设置。

• 删除：删除后不可恢复。

• 移动：移动到其他相册。

3.1.3 隐私设置（核心）

每张照片和每个相册均支持以下隐私级别（单选）：

隐私级别 可见范围 适用场景

仅自己可见 仅本人 私人回忆

全族公开 本家族所有成员（含游客模式下的本家族游客） 默认推荐

上下几代公开 本人的直系祖先（向上3代）+ 直系后代（向下3代） 家庭内部回忆

平台公开 全平台注册用户（需登录） 愿意分享给所有人

同地点用户公开 在相同地点（同一村寨/街道）生活过的用户 寻找同乡

隐私继承规则：
• 新建照片时，默认继承所在相册的隐私设置。

• 单张照片可独立修改隐私设置，覆盖相册级设置。

• 修改相册隐私设置时，可选择“同时修改所有照片”或“仅修改后续上传的照片”。

3.2 留言板（类似微博客）

3.2.1 发布留言

• 入口：用户中心 → 个人空间 → 留言板 → 输入框。

• 内容限制：纯文本，最多 200 字。

• 可选配图：最多 1 张图片（与相册共享 200MB 容量）。

• 隐私设置：与相册相同的五个级别（仅自己可见 / 全族公开 / 上下几代公开 / 平台公开 / 同地点用户公开）。

3.2.2 留言列表

• 按发布时间倒序排列。

• 每条留言显示：头像、姓名、发布时间、内容、配图（如有）、隐私级别图标。

• 支持按年份筛选。

3.2.3 留言操作

• 编辑：发布后 30 分钟内可编辑内容，超过后不可修改。

• 删除：随时可删除自己的留言。

• 点赞：其他家族成员可点赞（显示点赞数，v1.1 实现）。

• 评论：其他家族成员可评论（v1.1 实现，本次仅展示留言本身）。

3.2.4 隐私设置

• 每条留言独立设置隐私级别（同相册的五级体系）。

• 发布后不可修改隐私级别（避免已可见的用户突然失去访问权限）。

4. 数据架构

4.1 新增表

-- 相册表
CREATE TABLE user_albums (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  name VARCHAR(20) NOT NULL,
  description VARCHAR(100),
  cover_media_id BIGINT NULL,
  default_privacy ENUM('self', 'clan', 'lineage', 'public', 'same_location') DEFAULT 'clan',
  photo_count INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 个人照片表（与 media_archive 分离，属于个人空间）
CREATE TABLE user_photos (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  album_id BIGINT NOT NULL,
  file_url VARCHAR(500) NOT NULL,
  thumbnail_url VARCHAR(500),
  location_name VARCHAR(200) NOT NULL,
  taken_year INT NOT NULL,
  taken_date DATE,
  description VARCHAR(100),
  privacy ENUM('self', 'clan', 'lineage', 'public', 'same_location') DEFAULT 'clan',
  tagged_person_ids JSON,                    -- 关联的家族成员ID列表
  file_size BIGINT,                          -- 文件大小（字节）
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 留言板表
CREATE TABLE user_messages (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  content VARCHAR(200) NOT NULL,
  image_url VARCHAR(500) NULL,
  privacy ENUM('self', 'clan', 'lineage', 'public', 'same_location') DEFAULT 'clan',
  like_count INT DEFAULT 0,
  is_edited BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户存储用量表
CREATE TABLE user_storage (
  user_id BIGINT PRIMARY KEY,
  used_bytes BIGINT DEFAULT 0,               -- 已用字节数
  quota_bytes BIGINT DEFAULT 209715200,      -- 配额 200MB
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


4.2 索引建议

• user_photos：索引 (user_id, album_id) 和 (privacy, location_name)。

• user_messages：索引 (user_id, created_at) 和 (privacy)。

6. 错误处理与边界条件

场景 处理方式

上传照片时未填写地点/年份 阻止上传，提示“请填写地点和年份”

个人相册容量超限 提示“已用 200MB，请删除旧照片或联系管理员扩容”

留言内容超过 200 字 实时计数器变红，超过后阻止提交

留言发布超过 30 分钟后编辑 编辑按钮变灰，提示“已超过可编辑时间”

隐私设置为“同地点用户公开”但用户无地点数据 提示“请先在个人资料中补充童年地点”

7. 非功能性需求

• 性能：相册列表加载 < 1s，照片大图加载 < 2s。

• 存储：个人相册照片存储在独立 OSS 目录，与家族时光长廊隔离。

• 安全：隐私设置严格执行后端校验，前端不可绕过。

8. 验收标准（DoD）

1. 用户可创建相册，上传照片时必须填写地点、年份、时间，上传后照片显示在相册中。
2. 相册容量显示正确，达到 200MB 后无法继续上传。
3. 照片隐私设置为“仅自己可见”时，其他家族成员看不到该照片。
4. 照片隐私设置为“上下几代公开”时，仅直系三代内的成员可查看。
5. 用户可发布留言（≤200字），支持配图，隐私设置生效。
6. 留言发布 30 分钟后编辑按钮失效。

9. 与现有模块的关系

模块 关系

时光长廊 个人相册照片独立存储，不与家族时光长廊互通（但可手动分享到时光长廊）

寻找小伙伴 “同地点用户公开”的照片和留言可被匹配算法利用

迁徙地图 个人相册中标注地点的照片可展示在迁徙地图的个人足迹层（v1.1）

照片标注 个人相册照片暂不支持标注功能（v1.1 可考虑打通）

10. 后续迭代计划

• v1.1：留言板支持评论和点赞；相册照片支持标注人物。

• v1.2：相册照片可一键分享到时光长廊（需家族管理员审核）。

• v1.3：个人空间主题装扮（背景图、个性签名等）。
