根脉云谱（GeneaSphere）平台管理后台需求文档 v1.0

文档版本：v1.0  

对应系统版本：GeneaSphere Platform v1.0  

面向对象：平台运营团队、超级管理员、开发团队  

前置依赖：家族管理员后台、用户系统、支付系统、短信服务  

1. 产品概述

1.1 背景

平台运营方需要对所有注册家族、用户、订单、内容进行统一监管，确保平台健康运行。家族管理员后台负责单个家族的内部事务，而平台管理后台负责全局管控、数据统计、风控审核、商业化配置等。

1.2 目标

• 提供全局数据看板，实时掌握平台规模（家族数、用户数、照片数、存储用量等）。

• 支持对家族进行审核、封禁、数据导出等操作。

• 管理用户账号，处理投诉、封号、重置密码等。

• 审核全平台敏感内容（涉政、涉黄、违规寻亲帖等）。

• 管理印刷订单、充值订单、退款处理。

• 配置平台级参数（短信单价、AI工具定价、免费额度等）。

2. 角色与权限

角色 权限范围

超级管理员 全部权限，不可被限制

运营管理员 内容审核、订单管理、数据查看，不可修改系统配置

财务管理员 订单退款、充值对账，不可查看用户隐私数据

审计员 仅可查看操作日志和数据统计，不可执行任何写操作

3. 功能模块

3.1 控制台（Dashboard）

3.1.1 核心指标

• 实时数据：总家族数、总用户数（活跃/沉默）、总照片数、总存储用量（TB）。

• 今日新增：新注册家族、新注册用户、新上传照片、新订单数。

• 待处理事项：待审核家族（新注册需人工审核）、待审核内容（敏感图片/帖子）、待处理退款申请。

• 收入概览：本月总收入（印刷+AI工具+短信充值）、环比增长率。

3.1.2 趋势图

• 近7天/30天用户增长曲线。

• 近7天/30天收入曲线。

3.2 家族管理

3.2.1 家族列表

• 展示所有注册家族，字段：家族ID、名称、管理员姓名、成员数、注册时间、状态（正常/冻结/待审核）。

• 支持按名称、状态、注册时间筛选。

• 点击进入家族详情页。

3.2.2 家族详情

• 基本信息：名称、简介、管理员联系方式（脱敏）、注册IP。

• 数据概览：成员数、照片数、存储用量、短信余额、印刷订单数。

• 操作：

  • 审核通过：将待审核家族标记为正常。

  • 冻结：禁止该家族所有成员登录和操作，数据保留。

  • 解冻：恢复家族正常状态。

  • 删除：彻底删除家族数据（需二次确认，并备份）。

  • 数据导出：导出该家族全部数据（ZIP压缩包）。

3.2.3 家族审核

• 新注册家族默认进入“待审核”状态。

• 审核项：家族名称是否合规、管理员身份是否真实（需上传身份证或手持照片，可选）。

• 审核操作：通过 / 驳回（填写理由）。

3.3 用户管理

3.3.1 用户列表

• 展示所有平台用户，字段：用户ID、昵称、手机号（脱敏）、所属家族、注册时间、最后登录时间、状态（正常/封禁）。

• 支持按手机号、昵称、家族名搜索。

3.3.2 用户详情

• 个人信息：昵称、手机号、邮箱（如有）、注册IP、登录设备。

• 关联家族：所属家族及角色。

• 操作日志：最近10条操作记录。

• 操作：

  • 封禁：禁止登录，可选封禁时长（永久/7天/30天）。

  • 解封：恢复登录。

  • 重置密码：生成临时密码，强制用户下次登录修改。

  • 删除账号：彻底删除用户数据（需关联家族管理员确认）。

3.4 内容审核（全平台）

3.4.1 影像审核

• 展示所有家族中被标记为“待审核”的照片/视频（基于AI自动识别或用户举报）。

• 筛选：按家族、上传时间、审核状态。

• 操作：通过 / 删除（违规内容）/ 忽略。

3.4.2 寻亲帖审核

• 展示全平台所有寻亲帖，支持按状态（待审核/已发布/已下架）筛选。

• 操作：通过 / 下架（填写理由）。

3.4.3 举报管理

• 用户举报的帖子、照片、评论列表。

• 处理：确认违规（删除内容并警告举报人）/ 驳回举报。

3.5 订单管理

3.5.1 印刷订单

• 展示所有印刷订单，字段：订单号、家族名称、下单人、金额、状态（待支付/印刷中/已发货/已完成/已退款）。

• 支持按状态、时间范围筛选。

• 操作：

  • 查看详情：订单内容、收货地址、物流单号。

  • 标记发货：录入物流单号。

  • 退款：发起退款（原路返回，记录原因）。

3.5.2 充值订单

• 展示所有短信充值记录，字段：家族名称、充值金额、赠送金额、支付方式、交易号、状态（成功/失败）。

• 支持对账：导出CSV与支付平台账单核对。

3.6 系统配置

3.6.1 定价管理

• 短信单价：设置国内短信每条价格（默认0.05元）。

• AI工具定价：设置各工具的单次消耗次数（如修复1次、动态化3次）。

• 免费额度：设置新用户每月免费次数（默认10次）。

• 印刷价格：设置不同规格族谱的基准价格（可后续细化）。

3.6.2 家族默认配置

• 新注册家族的默认设置：每日短信上限、成员月接收上限、游客可见范围默认值。

3.6.3 全局开关

• 注册审核：开启后新家族需人工审核；关闭后自动通过。

• 短信功能：开启/关闭全平台短信发送功能（维护时使用）。

• AI工具：开启/关闭全平台AI工具箱。

3.7 数据统计与分析

3.7.1 报表中心

• 日报/周报/月报：自动生成PDF报表，包含核心指标和趋势。

• 家族排行：按成员数、照片数、存储用量、消费金额排行。

• 工具使用排行：各AI工具使用次数统计。

3.7.2 导出

• 支持导出各类统计数据为Excel/CSV。

3.8 日志审计

3.8.1 操作日志

• 记录所有平台管理员的操作：时间、操作人、操作类型、对象、详情。

• 支持按操作人、时间范围搜索。

• 日志不可删除，保留至少1年。

3.8.2 登录日志

• 记录平台管理员的登录记录：时间、IP、设备、成功/失败。

4. 数据架构

4.1 新增表

-- 平台管理员表
CREATE TABLE platform_admins (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role ENUM('super', 'operator', 'finance', 'auditor') DEFAULT 'operator',
  real_name VARCHAR(50),
  phone VARCHAR(20),
  status ENUM('active', 'disabled') DEFAULT 'active',
  last_login_at TIMESTAMP NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 平台操作日志表
CREATE TABLE platform_operation_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  admin_id BIGINT NOT NULL,
  action_type VARCHAR(50) NOT NULL,        -- 如 'review_family', 'ban_user', 'refund_order'
  target_type VARCHAR(50),                  -- 'family', 'user', 'order'
  target_id BIGINT,
  detail JSON,
  ip_address VARCHAR(45),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 全局配置表
CREATE TABLE global_settings (
  setting_key VARCHAR(100) PRIMARY KEY,
  setting_value JSON NOT NULL,
  description VARCHAR(200),
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


4.2 索引建议

• platform_operation_logs：索引 (admin_id, created_at) 和 (action_type)。

6. 错误处理与边界条件

场景 处理方式

超级管理员误操作（如误冻结家族） 提供操作日志回滚入口（24小时内可撤销）

待审核内容积压超过1000条 自动通知运营管理员，并标记优先级

退款金额大于订单金额 禁止操作，提示“退款金额不能超过订单金额”

平台管理员账号被锁定 超级管理员可手动解锁

7. 非功能性需求

• 性能：列表页加载 < 2s，数据统计报表生成 < 5s。

• 安全：所有接口需校验管理员Token和角色；敏感操作需二次确认。

• 可扩展性：配置项支持动态新增，无需发版。

8. 验收标准（DoD）

1. 超级管理员登录后可看到控制台数据，待处理事项准确。
2. 可搜索并查看任意家族详情，执行冻结/解冻操作后，该家族成员无法登录。
3. 可审核新注册家族，通过后家族状态变为正常。
4. 可查看全平台待审核内容，执行删除操作后内容从用户端消失。
5. 可查看印刷订单并标记发货，可发起退款。
6. 修改短信单价后，家族管理员端的费用计算随之更新。

9. 后续迭代计划
