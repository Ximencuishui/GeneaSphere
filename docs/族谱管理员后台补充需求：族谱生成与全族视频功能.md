族谱管理员后台补充需求：族谱生成与全族视频功能

文档版本：v2.1  

对应系统版本：寻根路 GeneaSphere Platform v1.0  

前置依赖：族谱核心数据、迁徙事件、大事件模块、AI视频生成服务、印刷模块  

面向对象：开发团队、测试团队、产品经理  

1. 族谱生成功能

1.1 功能概述

家族管理员可以将本家族的族谱数据（人物、关系、字辈、照片、生平）生成可视化的族谱文档，支持在线预览和印刷输出。生成的族谱可以是传统悬挂式（世系图）或现代图文混排版式，满足不同家族的需求。

1.2 用户故事

• 作为族长：我想把电子族谱打印成纸质版，分发给各房支，方便老人翻阅。

• 作为管理员：我想生成一份包含照片和生平的现代族谱，作为家族纪念册。

1.3 功能列表

1.3.1 生成参数设置

• 范围选择：

  • 全族（所有成员）

  • 指定房支（如长房、二房）

  • 指定世代范围（如第10代至第15代）

• 内容包含（多选）：

  • 人物基本信息（姓名、生卒、字辈）

  • 配偶信息

  • 子女列表

  • 生平简介（从 bio_text 提取）

  • 照片（优先使用正式肖像，若无则使用占位符）

  • 迁徙记录（每个人物的迁徙轨迹）

• 排版风格：

  • 传统悬挂式（世系图，适合大幅挂轴）

  • 现代图文混排（A4册子，照片+文字）

  • 简约列表式（适合快速查看）

• 封面设置：上传封面图片、输入书名（如“张氏族谱·2026年版”）、编纂日期。

1.3.2 生成与预览

• 点击“生成预览”后，系统异步生成族谱文档（PDF格式），生成过程中显示进度。

• 生成完成后，在线预览（使用PDF预览组件，支持翻页）。

• 管理员可在线编辑文字内容（如修改生平、添加注释），编辑后重新生成。

1.3.3 印刷下单

• 预览满意后，点击“下单印刷”进入印刷模块，自动带入页数、规格（默认A4精装，可改）。

• 支持一本起订，价格按页数计算。

1.3.4 历史版本管理

• 每次生成的族谱文档保存为历史版本，管理员可查看、下载、删除。

• 支持对比两个版本的差异（如人物增减）。

1.4 数据来源

• 人物数据：persons 表

• 关系数据：family_units + family_children + person_ancestry（闭包表）

• 照片：media_archive 中关联的人物照片（优先使用 is_portrait 标记的）

• 生平：bio_text 字段

• 迁徙：migration_events 表

1.5 验收标准

1. 管理员选择全族范围，生成预览后得到一份包含所有在世和已故成员的族谱PDF。
2. 选择“传统悬挂式”，生成的世系图正确显示父子、夫妻关系。
3. 选择“现代图文混排”，每位成员的照片和生平正确排版。
4. 下单印刷后，订单信息正确传递到印刷模块。
5. 历史版本列表中可查看之前生成的族谱文档。

2. 全族迁徙历史视频生成

2.1 功能概述

基于全族的迁徙事件数据，生成一段动态视频，展示家族从起源地到现今的迁徙历程。视频可自定义时间范围（如从明朝至今），并包含地图动画、关键地点照片、时间轴和文字解说。

2.2 用户故事

• 作为族长：我想在祭祖大会上播放一段视频，展示我们家族几百年来从江西迁到四川的历程。

• 作为管理员：我想生成一段包含老照片的迁徙视频，发到家族群里让大家了解家族历史。

2.3 功能列表

2.3.1 参数设置

• 时间范围：起始年份 ~ 结束年份（默认全族最早迁徙事件至最新事件）。

• 包含支系：全族 / 指定房支（可选）。

• 展示内容：

  • 地图动画（POI点闪烁、路径流动）

  • 关键地点照片（从 media_archive 中提取与该地点相关的照片）

  • 文字解说（自动生成迁徙原因、人数等）

  • 背景音乐（可选，提供预设曲库）

• 视频风格：怀旧（泛黄滤镜）/ 现代（明亮清晰）/ 庄重（黑白+慢速）。

2.3.2 生成流程

1. 系统从 migration_events 表中提取指定时间范围内的所有迁徙事件。
2. 按时间排序，生成路径点序列。
3. 为每个地点匹配关联照片（从 media_archive 中 taken_location 匹配）。
4. 调用AI视频生成服务（复用历史音像墙的管道），生成MP4视频。
5. 生成完成后，管理员可预览、下载、分享。

2.3.3 视频内容结构（示例）

• 开场：家族名称、迁徙总览地图

• 分段：每个迁徙节点展示：

  • 地图自动缩放至该地点

  • 显示迁出地→迁入地的流动路径

  • 该地点的老照片以光晕效果浮现

  • 文字解说（如“康熙年间，张氏先祖从福建汀州迁至四川重庆府”）

• 结尾：家族当前分布图，总结语

2.4 数据来源

• 迁徙事件：migration_events 表

• 地点照片：media_archive 中 taken_location 匹配

• 人物：persons 表（关联迁徙事件中的人物）

• 朝代图层：historical_dynasties 表（用于背景地图切换）

2.5 验收标准

1. 管理员选择时间范围“1600-1900”，生成的视频包含该时间段内的所有迁徙事件。
2. 视频中地图动画正确显示路径流动，POI点有光晕效果。
3. 每个迁徙节点至少展示一张关联照片（若有）。
4. 视频时长在合理范围内（每分钟约展示2-3个事件）。
5. 支持下载MP4格式，分辨率1080p。

3. 全族大事件视频生成

3.1 功能概述

家族大事件包括祭祖、修谱、建祠、重要人物诞辰/逝世、家族聚会等。管理员可以选择时间范围，将这些事件串联成一段视频，展示家族的重要历史时刻。

3.2 前置条件：大事件管理模块（需新增）

3.2.1 大事件管理（在后台“系统设置”下新增）

• 事件列表：展示所有已记录的大事件，字段：事件名称、时间（年月日或年份）、地点、描述、关联照片、事件类型（祭祖/修谱/建祠/诞辰/逝世/聚会/其他）。

• 新增/编辑事件：管理员手动录入，支持上传照片。

• 批量导入：支持CSV导入（用于从旧族谱中批量录入）。

• 自动生成：系统可基于族谱数据自动生成部分事件（如人物诞辰/逝世，需管理员确认）。

3.2.2 数据表（新增）

CREATE TABLE family_events (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  clan_id BIGINT NOT NULL,
  event_name VARCHAR(100) NOT NULL,
  event_type ENUM('ancestor_worship', 'genealogy', 'building', 'birth', 'death', 'gathering', 'other'),
  event_date DATE,
  event_year INT,                          -- 若只有年份
  location VARCHAR(200),
  description TEXT,
  media_ids JSON,                          -- 关联的照片ID列表
  created_by BIGINT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


3.3 视频生成功能

3.3.1 参数设置

• 时间范围：起始年份 ~ 结束年份（默认全族最早事件至最新事件）。

• 事件类型筛选：全选 / 指定类型（如只选“祭祖”和“修谱”）。

• 展示内容：照片幻灯片 + 文字描述 + 背景音乐。

• 视频风格：同迁徙视频。

3.3.2 生成流程

1. 按时间排序事件。
2. 每个事件展示1-3张关联照片（若无照片则使用文字卡片）。
3. 文字描述显示事件名称、时间、地点、简要描述。
4. 生成视频，时长按事件数量自动调整（每个事件约5-10秒）。

3.4 验收标准

1. 管理员录入3个测试事件（祭祖、修谱、聚会），生成视频后按时间顺序正确展示。
2. 每个事件的照片和描述正确对应。
3. 视频可下载和分享。
