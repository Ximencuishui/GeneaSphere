3.3 发送控制

3.3.1 立即发送

• 点击“立即发送”后，系统首先校验家族余额是否足以支付本次发送费用。

• 若余额不足，弹出充值提示，引导管理员充值后再发送。

• 余额充足则直接提交至短信服务商，并从余额中扣除对应费用。

3.3.2 定时发送

• 支持选择未来的日期和时间（精确到分钟）。

• 定时任务在指定时间自动执行，执行前会再次校验余额是否充足（防止期间余额被其他操作消耗）。

• 余额不足时，定时任务自动暂停，并向管理员发送站内信通知。

3.4 发送记录与统计

3.4.1 发送列表

• 展示所有历史发送记录，字段：发送时间、内容摘要、收件人数、成功数、失败数、费用（元）、状态（已发送/定时中/失败/余额不足）。

• 支持按时间范围、状态筛选。

3.4.2 详情

• 点击某条记录查看详情：完整内容、收件人列表（脱敏显示手机号中间四位）、失败原因、扣费明细。

3.4.3 费用统计

• 显示本月累计发送条数、累计消费金额、当前余额。

• 家族管理员可设置“余额低水位预警”（如低于 20 元时发送站内信通知管理员）。

4. 计费与余额钱包（新增章节）

4.1 计费标准

项目 价格 说明

国内短信 0.05 元/条 按实际成功发送条数扣费（失败不扣费）

国际短信 暂不支持 后续迭代

注：实际单价可根据运营商成本浮动，平台保留调整权，调整前会公示。

4.2 余额钱包机制

4.2.1 充值

• 入口：家族管理员后台 → “短信通知” → “余额管理”。

• 充值方式：支持微信支付、支付宝。

• 充值档位：

  • 50 元（赠送 0 元）

  • 100 元（赠送 10 元）

  • 200 元（赠送 30 元）

  • 500 元（赠送 100 元）

  • 自定义金额（无赠送）

• 到账：支付成功后余额立即更新。

4.2.2 扣费

• 发送短信时，按实际成功条数 × 单价从余额中扣除。

• 扣费时机：短信发送完成后，根据运营商回执确认成功条数后进行扣费（异步，通常在发送后几分钟内）。

• 扣费记录在 sms_cost_logs 表中，与发送记录一一对应。

4.2.3 余额查询

• 管理员后台首页展示当前余额。

• 发送页面顶部显示余额，若余额低于 10 元，显示黄色警告条。

4.2.4 余额预警

• 管理员可设置预警阈值（默认 20 元）。

• 当余额低于阈值时，系统自动向管理员发送站内信提醒。

• 当余额为 0 时，所有发送操作被阻止，直到充值。

4.2.5 退款

• 家族解散或平台停止服务时，可申请退款剩余余额。

• 退款规则：扣除手续费（5%）后退回原支付账户，7个工作日内到账。

4.3 数据表变更

新增表

-- 家族余额表
CREATE TABLE clan_balances (
  clan_id BIGINT PRIMARY KEY,
  balance DECIMAL(10,2) DEFAULT 0.00,        -- 当前余额（元）
  total_recharged DECIMAL(10,2) DEFAULT 0.00, -- 累计充值总额
  total_consumed DECIMAL(10,2) DEFAULT 0.00,  -- 累计消费总额
  low_balance_threshold DECIMAL(10,2) DEFAULT 20.00, -- 低水位预警阈值
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 充值记录表
CREATE TABLE recharge_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  clan_id BIGINT NOT NULL,
  operator_id BIGINT NOT NULL,                -- 操作管理员用户ID
  amount DECIMAL(10,2) NOT NULL,              -- 充值金额（不含赠送）
  bonus DECIMAL(10,2) DEFAULT 0.00,           -- 赠送金额
  payment_method VARCHAR(20),                 -- wechat / alipay
  trade_no VARCHAR(100),                      -- 支付平台交易号
  status ENUM('pending', 'success', 'failed') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 扣费记录表
CREATE TABLE sms_cost_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  record_id BIGINT NOT NULL,                  -- 关联 sms_send_records.id
  clan_id BIGINT NOT NULL,
  success_count INT NOT NULL,
  unit_price DECIMAL(10,4) NOT NULL,          -- 单价（元/条）
  total_cost DECIMAL(10,2) NOT NULL,          -- 总费用
  deducted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


修改 sms_send_records 表

• 增加字段 cost_status ENUM('pending', 'deducted', 'refunded') DEFAULT 'pending'，标记扣费状态。

6. 错误处理与边界条件

场景 处理方式

余额不足时点击发送 弹出“余额不足”提示，显示当前余额和所需费用，提供“立即充值”按钮

定时发送时余额不足 定时任务暂停，发送站内信通知管理员，任务保留24小时后自动取消

充值支付失败 订单标记为失败，余额不变，用户可重新发起充值

运营商扣费与实际不符 平台定期对账，差额部分在下一次充值时调整（多退少补）

家族余额长期为0（超过90天） 自动发送提醒，若仍不充值则暂停该家族的短信功能

7. 非功能性需求

（同 v1.0，补充以下条目）

• 对账：每日凌晨自动生成对账单，与运营商账单核对，确保扣费准确。

• 安全：充值操作需记录操作人 IP，防止盗刷。

8. 验收标准（DoD）

1. 管理员首次进入发送页面，余额显示为 0，点击发送时弹出充值提示。
2. 管理员充值 100 元（到账 110 元含赠送），余额显示 110 元。
3. 发送一条短信给 10 人，成功后余额减少 0.5 元（10 × 0.05），扣费记录正确。
4. 余额低于 20 元时，管理员收到预警站内信。
5. 定时发送时余额不足，任务暂停，管理员收到通知。
