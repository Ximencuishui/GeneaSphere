# 2. 启动前端

| # | 验收项 | 实现位置 | 状态 |

## 三、补缺模块验收（P0 / P1）

### 3.1 登录失败锁定

| 检查 | 状态 | 说明 |
| --- | --- | --- |
| 连续 5 次错误密码触发锁定 | ✅ | `LoginLockService.recordFailure()` 在第 5 次设置 `locked_until = now + 30min` |
| 第 6 次返回 401 + 锁定消息 | ✅ | `LoginLockService.checkLock()` 抛 `UnauthorizedException` |
| 登录成功后失败计数清零 | ✅ | `LoginLockService.clearFailures()` |
| 锁定窗口过期后自动允许重试 | ✅ | `checkLock()` 检测过期则重置 `failed_count = 0` |
| 平台端和家族端共享同一机制 | ✅ | 通过 `subject_type: USER / PLATFORM_ADMIN` 区分 |
| 演示账号不参与锁定 | ✅ | `auth.service.ts: demoLogin/demoMemberLogin` 主动 `clearFailures` |
