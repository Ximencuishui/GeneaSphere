# 寻根路 Admin 后台 - 日志保留策略

> 适用范围：`AuditLog`（家族管理员操作日志）与 `PlatformOperationLog`（平台管理员操作日志）
> 对应需求：《寻根路 Admin 管理后台需求文档 v1.0》§3.10

## 1. 总体策略

- **保存期限**：所有操作日志在生产数据库内保留 **至少 1 年**，1 年后可迁移至冷存储（如对象存储归档桶）长期保留。
- **不可篡改**：日志表不提供删除 / 修改接口（已通过后端 Service 验证：无 `delete` / `update` 控制器方法）。
- **不可删除**：即便超级管理员账号也无法通过 API 删除日志。物理删除需 DBA 通过数据库后台手动操作（不在产品功能范围）。
- **追加写入**：日志通过 Prisma `create` 接口写入，写入失败时由调用方捕获并记录，不影响主业务。

## 2. 数据模型

### AuditLog（家族管理员操作日志）

```prisma
model AuditLog {
  id          BigInt   @id @default(autoincrement())
  clan_id     BigInt?
  user_id     String
  action      String
  target_type String?
  target_id   String?
  details     String?
  ip_address  String?
  created_at  DateTime @default(now())

  @@index([clan_id, created_at])
  @@index([user_id])
  @@index([action])
}
```

### PlatformOperationLog（平台管理员操作日志）

```prisma
model PlatformOperationLog {
  id          BigInt   @id @default(autoincrement())
  admin_id    BigInt
  action_type String
  target_type String?
  target_id   String?
  detail      Json?
  ip_address  String?
  status      String  @default("success")
  created_at  DateTime @default(now())

  @@index([admin_id, created_at])
  @@index([action_type])
  @@index([target_type])
}
```

## 3. 索引设计

- `AuditLog(clan_id, created_at)`：支撑家族管理员后台日志按家族 + 时间区间查询
- `AuditLog(user_id)`：支撑用户维度的操作轨迹
- `AuditLog(action)`：支撑按操作类型筛选
- `PlatformOperationLog(admin_id, created_at)`：支撑平台管理员操作记录
- `PlatformOperationLog(action_type)`：支撑按操作类型汇总

## 4. 数据归档（建议）

```sql
-- 1. 导出 1 年前的日志到冷存储（OSS / S3）
COPY (
  SELECT * FROM audit_logs
  WHERE created_at < NOW() - INTERVAL '1 year'
) TO '/tmp/audit_logs_archive.csv' CSV HEADER;

-- 2. 导出后从数据库删除
DELETE FROM audit_logs
WHERE created_at < NOW() - INTERVAL '1 year';
```

> ⚠️ 归档操作由 DBA 手工执行，**不**提供 API 入口。

## 5. 应急解锁（误锁账号）

平台管理员账号连续 5 次密码错误会被锁定 30 分钟。超级管理员可通过数据库手动解锁：

```sql
DELETE FROM login_attempts
WHERE subject_type = 'PLATFORM_ADMIN'
  AND subject_key = '被锁用户名';
```

## 6. 验收检查点

| 检查项 | 验证方式 | 期望结果 |
| --- | --- | --- |
| 后端无日志删除 API | grep 后端代码 `auditLog\.delete\|operationLog\.delete` | 无匹配 |
| 后端无日志修改 API | grep 后端代码 `auditLog\.update\|operationLog\.update` | 无匹配 |
| 后端所有关键操作均记录日志 | 单元测试覆盖 | 通过 |
| 索引存在 | `EXPLAIN` SQL 查询 | 命中索引 |
| 数据库保留时长 ≥ 1 年 | 查询 `MIN(created_at)` | 距今不超过 365 天 |
