1. 产品背景与目标

1.1 背景

家族规模扩大后，数据录入权限分散，导致族谱质量下降；同时隐私泄露风险增加。管理员需要一个集中管控平台，既能保证数据准确性，又能灵活控制信息可见范围。

1.2 目标

• 数据治理：审核所有UGC（照片、生平），防止错误或不当内容入库。

• 隐私合规：精细配置在世人员、游客的访问权限，满足不同家族的安全诉求。

• 寻亲归宗：安全、可控地接纳外支系，完成数据合并。

• 运营支撑：管理存储、印刷订单、操作审计，保障平台可持续发展。

2. 角色定义

角色 权限范围 典型用户

超级管理员（Owner） 全部权限，可转让所有权 开基祖、主要出资人

管理员（Admin） 内容审核、寻亲处理、系统配置、成员管理（不含删除Owner） 家族理事会成员

编辑者（Editor） 增删改人员信息、上传照片、修改个人生平 热心族人、各房代表

观察员（Viewer） 仅查看族谱、时光长廊，无编辑权限 年长族人、监事会

游客（Visitor） 仅查看公开信息（受隐私配置限制） 未登录或未验证的外人

4. 详细功能需求（含交互细节）

4.1 控制面板（Dashboard）

用户故事：作为管理员，我希望一登录就看到待办事项和关键数据，以便快速安排工作。

功能点：
• 统计卡片：总人数、在世人数、照片总数、存储用量（百分比）、待审核影像数、待处理寻亲申请数。

• 待办列表：最近5条待审核内容，点击直接跳转审核页面。

• 快速入口：一键跳转“隐私配置”、“字辈管理”。

交互规则：
• 统计卡片支持点击，进入对应列表页。

• 待办列表为空时显示“暂无待办事项，一切井然有序”。

4.2 成员列表与权限分配

用户故事：作为管理员，我需要查看所有族人，并能调整他们的角色，确保合适的人有合适的权限。

功能点：
1. 成员列表：
   • 展示字段：头像、姓名、性别、世代、房支、角色、最后活跃时间。

   • 支持按世代、房支、角色筛选；支持关键字搜索。

   • 点击姓名跳转至该成员的“个人详情页”（前台页面）。

2. 权限分配：
   • 点击“权限”按钮，弹出对话框。

   • 下拉选择角色：Owner/Admin/Editor/Viewer。

   • 敏感操作：将成员从Admin降级为Viewer，需二次确认（输入当前管理员密码）。

   • 转让Owner：仅Owner可操作，需输入新Owner的用户名并确认，原Owner自动降为Admin。

3. 批量操作：支持勾选多个成员，批量修改角色（仅限非Owner）。

边界条件：
• 不可将最后一个Admin降级为Viewer（需先指派新Admin）。

• 删除成员（移除出家族）需先确认该成员无子女关联，否则提示“请先转移子女关系”。

4.3 内容审核中心

用户故事：作为管理员，我需要审核族人上传的照片和生平，防止错误信息或隐私泄露。

4.3.1 影像审核

列表：
• 展示待审核影像的缩略图、上传者、上传时间、拍摄地点（若有）。

• 支持按时间排序、按上传者筛选。

• 缩略图鼠标悬停放大预览。

操作：
• 通过：照片进入对应相册（个人或家族时光长廊）。

• 驳回：弹出理由输入框（必填），可选择预设理由：“涉及隐私”、“图像模糊”、“重复上传”、“其他”。驳回后通知上传者。

• 设为封面：仅当照片被标记为“村寨风貌”或“祭祀活动”时，管理员可将其设为“聚落封面”（用于聚落页面展示）。

批量操作：支持勾选多条，批量通过/驳回。

4.3.2 生平审核

列表：展示待审核的生平标题、摘要、作者、提交时间。
详情：点击查看全文，支持富文本渲染。
操作：通过/驳回（驳回需填写理由）。

4.4 寻亲与归宗管理

用户故事：作为管理员，我希望能安全地处理外族人的认亲申请，借助系统比对信息做出判断，并一键完成数据合并。

4.4.1 认亲申请列表

展示字段：申请人用户名、自称祖籍地、字辈、关联祖先（系统推测）、申请时间、状态（待处理/已通过/已拒绝）。
筛选：按状态、按时间。

4.4.2 申请详情与比对工具

点击申请进入详情页：
• 左侧：申请人提供的支系信息（祖籍、字辈、关键祖先名、迁徙史）。

• 右侧：系统自动匹配的本家族相关节点，高亮显示重叠信息（如相同字辈、相似地名）。

• 比对评分：系统给出匹配度（百分比），并列出匹配依据。

操作：
• 通过：弹出“归宗确认”对话框。

  • 管理员需指定挂载点（选择本家族中的一位祖先作为父节点）。

  • 系统自动执行数据合并（闭包表重算），生成合并日志。

  • 合并后，申请人及其支系自动成为本家族成员，角色默认为“编辑者”。

• 拒绝：填写拒绝理由，系统通知申请人。

• 标记为“需人工核查”：当系统无法确定时，管理员可标记，后续线下沟通。

安全措施：
• 合并前自动创建家族数据快照，支持一键回滚（24小时内）。

• 合并操作记录到审计日志。

4.4.3 寻亲帖管理

管理员可查看本家族发布的寻亲帖，支持编辑、下架、置顶操作。

4.5 系统设置

用户故事：作为管理员，我需要灵活配置隐私规则，确保家族信息安全；同时维护字辈序列，保证数据规范性。

4.5.1 隐私配置

配置项：
1. 游客可见范围：
   • 开关：[ ] 允许游客查看已故人员信息

   • 数字输入：仅展示前 N 代（默认5，范围1-20）

   • 开关：[ ] 隐藏在世人员照片

   • 开关：[ ] 隐藏在世人员配偶信息

2. 亲属验证登录：
   • 启用开关：[ ] 开启亲属验证

   • 验证问题配置（多选）：

     ◦ ☑ 请输入您父亲的姓名

     ◦ ☑ 请输入您祖父的出生年份

     ◦ ☐ 请输入您曾祖父的配偶姓氏

   • 容错次数：输入框（默认3次）

3. 数据导出：一键导出家族全部数据（含照片）的压缩包，用于备份。

交互：保存后即时生效，前端游客访问时会立即应用新规则。

4.5.2 字辈管理

• 展示当前字辈序列（如：宏-道-永-正-兴-毓）。

• 支持编辑：管理员可按世代顺序添加/删除/修改字辈。

• 校验规则：新增人员时，系统根据其世代自动匹配字辈，若姓名不含对应字辈则发出警告（不强制阻止，但记录日志）。

• 历史字辈：支持记录多套字辈（如某房分支另立字辈），并标注适用范围。

4.5.3 云存储

• 展示当前存储用量（饼图：照片/视频/其他）。

• 显示剩余容量、已用百分比。

• 提供“扩容”按钮，跳转至付费页面（对接支付系统，本次暂不实现，仅展示提示）。

4.6 印刷订单管理

用户故事：作为管理员，我需要查看本家族的印刷订单状态，方便跟进。

列表：订单编号、下单时间、印刷规格（开本/纸张/装订）、数量、金额、状态（待支付/印刷中/已发货/已完成）、物流单号。
操作：点击订单号查看详情；已发货订单可查看物流轨迹（对接快递100 API）。
再次购买：支持一键复刻上次订单配置，重新下单。

4.7 日志审计

用户故事：作为管理员，我希望能够追溯关键操作，防止恶意破坏或误操作。

列表：展示操作日志，字段：时间、操作人、操作类型、对象、详情、IP地址。
操作类型：修改关系、删除人员、通过审核、驳回审核、合并支系、修改隐私配置、修改角色等。
搜索：支持按操作人、操作类型、时间范围搜索。
导出：支持导出CSV格式日志。

数据保留：日志保留至少180天，超期自动归档。

5. 非功能性需求

类别 要求

性能 列表页（成员、审核、日志）首次加载 < 2s，翻页 < 1s；操作响应 < 500ms。

安全 所有接口需校验 clan_id 与当前管理员所属家族一致（多租户隔离）；敏感操作需二次确认；日志不可篡改（仅追加）。

可用性 界面符合 Element Plus 设计规范，操作路径不超过3步；移动端适配良好。

兼容性 支持 Chrome/Firefox/Edge 最新两个版本。

可扩展性 权限模型支持未来新增角色；审核类型可动态扩展（如增加“视频审核”）。

7. 验收标准（DoD）

1. 权限分配：Owner可将任一成员设为Admin，Admin不可将Owner降级；敏感操作需二次确认。
2. 内容审核：上传一张测试照片，管理员可在审核列表看到，通过后照片出现在时光长廊，驳回后上传者收到通知。
3. 隐私配置：设置“游客仅看5代以上”，退出登录后访问族谱树，在世人员信息被隐藏，已故人员可见。
4. 归宗合并：模拟一个外支系申请，管理员通过比对工具确认后合并，合并后数据正确，且操作日志可查。
5. 日志审计：所有关键操作（修改关系、删除人员、合并支系）均记录日志，且不可删除。
6. 性能：在5000成员、10000张照片的测试数据下，列表页加载不超过2秒。
