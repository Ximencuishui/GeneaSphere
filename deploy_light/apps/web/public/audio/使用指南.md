## 🚀 快速开始

### 1. 准备音频资源

在测试前，需要先添加音频文件到 `apps/web/public/audio/` 目录：

#### 方式一：使用免费音源（推荐）

1. 访问 [Freesound.org](https://freesound.org)
2. 搜索关键词：
   - `crickets night` (虫鸣)
   - `frog chorus` (蛙声)
   - `summer evening ambience` (夏日夜晚)
3. 筛选 License: **Creative Commons 0** (公共领域)
4. 下载 MP3 格式文件
5. 重命名并放置到 `apps/web/public/audio/` 目录

#### 方式二：使用占位文件测试

暂时可以使用任何 MP3 文件重命名为：
- `night-crickets.mp3`
- `field-frogs.mp3`
- `summer-night.mp3`

### 2. 启动开发服务器

```bash
cd apps/web
pnpm dev
```

### 3. 访问页面

打开浏览器访问：`http://localhost:5173`

## 🎮 使用方式

### 基础操作

1. **开启背景音乐**
   - 点击页面右下角的"🔇 背景音乐"按钮
   - 控制面板自动展开，音乐开始播放

2. **播放控制**
   - 点击"▶ 播放" / "⏸ 暂停"按钮切换播放状态
   - 或在浮动按钮上直接点击切换

3. **调节音量**
   - 拖动音量滑块（0-100%）
   - 实时生效

4. **切换音效**
   - 在"音效"下拉菜单中选择不同的音效
   - 当前播放会自动切换到新音效

5. **定时关闭**
   - 选择定时时间：15分钟 / 30分钟 / 1小时 / 2小时 / 不关闭
   - 到达时间后自动淡出停止（2秒渐弱）
   - 显示剩余时间倒计时

### 用户偏好保存

系统自动保存以下设置到浏览器本地存储：
- 音量大小
- 选择的音效
- 定时关闭时间

刷新页面后自动恢复上次的设置。

## ✨ 功能特性

### 1. 无缝循环播放
- 音频自动循环，无间断
- 使用 HTML5 Audio API 的 `loop` 属性

### 2. 淡出效果
- 定时关闭时，2秒内音量逐渐降低至0
- 避免突然停止带来的不适感

### 3. 状态指示
- 播放中按钮变为粉色渐变
- 底部显示动态音波动画
- 图标从 🔇 变为 🎵

### 4. 响应式设计
- 桌面端：固定在右下角
- 移动端：自适应屏幕宽度
- 支持触摸操作

### 5. 错误处理
- 音频加载失败时显示友好提示
- 浏览器不支持时降级处理
- 网络断开时的容错机制

## 🐛 常见问题

### Q1: 点击播放没有声音？

**原因**：音频文件未找到或浏览器限制自动播放

**解决方案**：
1. 检查 `public/audio/` 目录下是否有对应的 MP3 文件
2. 确保用户有交互操作（点击按钮）后才播放
3. 查看浏览器控制台是否有错误信息

### Q2: 音效无法循环？

**原因**：音频文件本身有静音段或循环点设置不当

**解决方案**：
1. 使用 Audacity 检查音频文件
2. 确保首尾可以无缝衔接
3. 使用 Crossfade 技术处理循环点

### Q3: 移动端无法播放？

**原因**：iOS/Android 对音频自动播放有严格限制

**解决方案**：
1. 确保用户点击后才开始播放
2. 不要在页面加载时自动播放
3. 测试时使用真实的用户交互触发

### Q4: 定时关闭没有生效？

**原因**：定时器逻辑异常或页面被切换

**解决方案**：
1. 检查浏览器控制台是否有 JavaScript 错误
2. 确保页面在前台运行
3. 切换回页面时检查定时器状态
